Mate-Choice Copying Across Animal Taxa

A pre-registered, systematic meta-analysis examining the prevalence and mechanisms of non-independent mate selection driven by social observation.

01 · Overview

Research Question

Mate-choice copying (MCC) occurs when an individual's preference for a potential mate is influenced by observing other individuals' choices. This project synthesises the quantitative evidence for MCC across animal taxa, tests for moderating variables (species, sex, experimental design), and explicitly models lab-level non-independence arising from "hub labs" that dominate this research area.

02 · Data Sources

Literature Search

Records were retrieved from two major bibliographic databases and combined in Rayyan format before import into R for deduplication.

🗃 Scopus

Searched with terms covering mate choice, copying, non-independent mate selection, and social information. Exported in RIS/CSV Rayyan format.

🗃 Web of Science

Parallel search with identical Boolean query. Duplicate entries across databases were resolved programmatically (see Pipeline section).

03 · Data Processing Pipeline

Deduplication & Assignment

All cleaning and assignment is handled by a single reproducible R script: scripts/0_literature_screening/1_deduplicate_records.R

1
R · data.table

Load & Inspect

Import combined_records_final.csv (2,083 rows, 22 columns) with data.table::fread(). Initial inspection flags records missing abstract, authors, or DOI for quality scoring in later steps.

2
R · stringr

Title-Based Deduplication

Titles are normalised to a canonical form before matching. Duplicate groups are resolved by selecting the highest-quality record.

# Normalise title
articles[, title_lwr := str_to_lower(title)]
articles[, title_lwr := gsub("[[:punct:]]", " ", title_lwr)]
articles[, title_lwr := gsub(" {2,}", " ", title_lwr)]
articles[, title_lwr := trimws(title_lwr)]

# Quality score (0–3): abstract + authors + DOI
articles[, quality := has_abstract + has_authors + has_doi]

# Keep highest-quality record per title group
articles.title.dedupe <- articles[
  , .SD[which.max(quality), ], by = .(title_lwr)
]
3
R · data.table

DOI-Based Deduplication

A second deduplication pass on the DOI field catches cases where titles differ slightly across databases but resolve to the same article. The helper function split_conc() uniquely concatenates source-file strings across all merged duplicates before selecting the best record.

split_conc <- function(x) {
  str_split(x, pattern = "; ") |>
    unlist() |> unique() |> sort() |>
    paste(collapse = "; ")
}

# Concatenate source files per DOI group, then keep highest quality
sources <- articles.title.dedupe.mrg[
  , .(source_file = split_conc(source_file)), by = .(doi)
]
articles.doi.dedupe <- articles.title.dedupe.mrg[
  , .SD[which.max(quality), ], by = .(doi)
]
4
R · Data decision

Year Filter: ≥ 2019

Following deduplication, only records published in or after 2019 are retained. This reflects the pre-registered scope: focusing on recent experimental literature where methodology is more standardised. The filter reduces the pool from ~2,000 deduplicated records to 495 articles.

articles.final <- articles.final[year >= 2019, ]
message("Earliest year in dataset: ", min(articles.final$year))
Data decision: The year ≥ 2019 cutoff was chosen to align with the pre-registration scope and to focus on studies reporting effect sizes in a format compatible with modern meta-analytic synthesis. Earlier literature is not excluded from the original database — it is retained in combined_records_final.csv and can be incorporated in sensitivity analyses.
5
R · Random assignment

Random Shuffle & Screener Assignment

The 495 articles are shuffled with sample(.N) to prevent ordering bias, then distributed to screeners using a dynamic assignment function that guarantees near-equal loads.

# Shuffle
articles.final <- articles.final[sample(.N), ]

# Dynamic assignment: ceiling(total / n_screeners) reps, trimmed to total
create_assignments <- function(names, total_rows) {
  reps_needed <- ceiling(total_rows / length(names))
  full_vector <- rep(names, each = reps_needed)
  return(full_vector[1:total_rows])
}

# Primary screeners (~33 articles each across 15 people)
articles.final[, primary_screener := create_assignments(primary_list, .N)]

# Secondary validators (Eduardo & Losia — all 495 articles split between them)
articles.final[, secondary_screener := create_assignments(secondary_list, .N)]
6
Output CSVs

Export Person-Specific Files

One CSV is written per primary screener to builds/0_literature_screening/deduplicated_and_randomly_split/. Each file contains only that screener's assigned articles together with both primary_screener and secondary_screener columns for cross-referencing.

05 · Data Extraction

NotebookLM AI Extraction Pipeline

Included articles are processed through a four-step AI-assisted extraction protocol in NotebookLM (Google). Each step uses a structured prompt grounded by a MetaAnalysis_Reference_Sheet.pdf data dictionary and few-shot examples. A fresh chat session is opened for each step ("cache-bust" rule) to prevent AI recycling of earlier outputs.

Operating rule — Cache Bust: Each of the four steps below must be run in a completely new NotebookLM chat window. This forces the model to re-read the PDFs freshly and prevents it from re-using tables generated in previous steps.
1
AI · Species ID

Species & Basic Metadata

For every included PDF, extract the study organism as a full binomial nomenclature (e.g., Poecilia reticulata). Study IDs are constructed as FirstAuthorLastName_Year_CamelCaseJournal (e.g., Belkina_2021_AnimalBehaviour). Common names alone are not accepted.

2
AI · Network

Co-Authorship & Lab Identification

Generates a long-format table — one row per author per paper — enabling construction of a bipartite co-authorship network. Each author row carries a standardised Lab Identifier in the format PILastName_Institution_City (e.g., Dukas_UniversityOfToronto_Toronto). The last author is designated as PI; this identifier is applied consistently to all co-authors on the same paper, linking the entire author list to one "primary lab" node.

This step feeds directly into network-based tests of lab-level non-independence. "Hub labs" with disproportionate publication output can inflate effect-size estimates if not modelled.
3
AI · Structure

Trial-Level Data Mapping

Maps the internal experimental structure of each paper: how many distinct trials exist, sample sizes, independence status, and the exact page/figure/table location of the relevant statistics. Key columns:

  • Subject Independence — "Independent Cohort" vs. "Repeated Measures"
  • Control Group Shared — Yes/No, with cross-reference to paired trial
  • Data Location — "Table 1, Page 5" or "Figure 3A, Page 8"
4
AI · Multi-Agent

Quantitative Data Extraction (3-Agent Pipeline)

The most critical step. Three simulated AI agents run sequentially per document to extract and cross-validate the actual numbers (means, SDs/SEs, counts, test statistics).

🔍 Agent 1 — Extractor

Scans Methods and Results to pull continuous data (Mean ± SD/SE, N), binary counts, or inferential statistics (F, t, χ²) for every mate-choice copying trial in the paper.

🧐 Agent 2 — Verifier

Acts as a skeptical methodologist. Checks Agent 1's extraction for three specific failure modes:

  • SD/SE confusion
  • Non-independence of sample sizes
  • Missing trials / secondary experiments
⚖️ Agent 3 — Arbiter

Resolves conflicts identified by Agent 2 and formats the validated data into the standardised output table, including verification notes and exact source locations.

06 · Planned Analyses

Statistical Analyses

The following analyses are planned following completion of the data extraction phase. All models will be implemented in R.

📐 Meta-Analytic Models

Multi-level random-effects meta-analysis to estimate the overall effect size for mate-choice copying, accounting for the nested structure of trials within papers within labs. Heterogeneity will be quantified with I² and τ².

🌐 Co-Authorship Network Analysis

Bipartite lab–paper network built from Step 2 extraction output. Hub labs are identified by degree centrality. Non-independence is modelled by including a lab-level random effect or using network-based weights.

🌳 Phylogenetic Comparative Methods

Species non-independence is controlled via a phylogenetic covariance matrix incorporated into the meta-analytic model (e.g., metafor + ape). This accounts for the possibility that closely related species share similar MCC propensity due to shared evolutionary history.

📉 Publication Bias

Funnel plot asymmetry, Egger's test, and trim-and-fill analysis. Time-lag bias will be examined by plotting effect size against publication year.

🔍 Moderator Analyses

Meta-regression models will test whether effect sizes vary by sex of the copying individual, taxonomic group, experimental design (laboratory vs. semi-natural), and model presentation method.

🔁 Sensitivity Analyses

Inclusion of pre-2019 records, leave-one-out analysis for influential studies, and re-analysis excluding studies from identified hub labs to assess robustness of the main estimate.
