Introduction & Project Update

This protocol implements the overall effect size synthesis for the updated Mate-Choice Copying (MCC) meta-analysis.

Specifically, we integrate:

  1. The New Extractions: Our newly extracted and verified dataset containing recent trials.
  2. Original Continuous Dataset (Davies et al. (2020)): Standardised mean differences (Hedges’ \(d\)) analyzing continuous metrics.
  3. Original Binary Dataset (Jones and DuVal (2019)): Log Odds Ratio (\(\ln\text{OR}\)) analyzing binary choices.

The R script aligns and synthesizes these three sources into unified datasets, allowing comparison of meta-analytic summaries across three key subsets:

  • New Extractions Only: Eduardo’s new dataset.
  • Original Only: Legacy data from the original publications.
  • Combined Dataset: A complete synthesis of old and new trials to estimate the updated global effect sizes.

Model Fitting Function

This helper function handles the variance–covariance (VCV) matrix construction and fits a multilevel random-effects model using rma.mv() in metafor.

  • Variance–Covariance: We construct a VCV matrix via vcalc() to handle dependencies among multiple effect sizes within the same study (assuming \(r = 0.5\) for nested measurements).

  • Random intercepts are modeled for:

    • Study (identifierStudyId)
    • Individual trial/effect size (identifierEffectSizeID)
    • Species (taxonomySpecies) — the non-phylogenetic species effect, capturing species-specific deviations not explained by shared ancestry
    • Phylogeny (species_phylo) — a phylogenetic random effect supplied with a species-by-species correlation matrix (R = list(species_phylo = phylo_cor)) derived from the Open Tree of Life topology with Grafen branch lengths (see scripts/2_phylogeny/build_phylogeny.R). This partitions the among-species variance into a component attributable to phylogenetic relatedness and a residual species component.

The phylogenetic term is added to the uncorrected models only (for the New, Original, and Combined datasets); the small-study bias-corrected (Egger-type) models retain the original structure. When a phylogenetic correlation matrix is unavailable, the fit gracefully falls back to the non-phylogenetic structure.

fit_overall_mv <- function(dat, yi = "yi", vi = "vi", cluster_var = "identifierStudyId", obs_var = "identifierEffectSizeID", rho = 0.5, phylo_cor = NULL, phylo_var = "species_phylo") {
    # Remove rows with missing values
    dat2 <- dat %>%
        filter(!is.na(.data[[yi]]), !is.na(.data[[vi]]), .data[[vi]] > 0)

    if (nrow(dat2) == 0) {
        message("No complete cases found.")
        return(NULL)
    }

    # 1. Build VCV Matrix
    VCV <- metafor::vcalc(
        vi = dat2[[vi]],
        cluster = dat2[[cluster_var]],
        obs = dat2[[obs_var]],
        rho = rho,
        data = dat2
    )

    # 2. Assemble random-effects structure. Always include study, effect-size,
    #    and a (non-phylogenetic) species intercept. If a phylogenetic
    #    correlation matrix is supplied AND every species level is present in it,
    #    add a phylogenetic random effect with R = subset-of-correlation-matrix.
    random_list <- list(
        ~ 1 | identifierStudyId,
        ~ 1 | identifierEffectSizeID,
        ~ 1 | taxonomySpecies
    )
    R_arg <- NULL
    used_phylo <- FALSE

    if (!is.null(phylo_cor) && phylo_var %in% colnames(dat2)) {
        sp <- unique(as.character(dat2[[phylo_var]]))
        missing_sp <- setdiff(sp, rownames(phylo_cor))
        if (length(missing_sp) == 0) {
            # Subset the correlation matrix to the species actually present so the
            # dimnames match the grouping levels exactly. Also drop any stale
            # (unused) factor levels on the grouping column so metafor's level
            # set matches the R-matrix dimnames.
            dat2[[phylo_var]] <- factor(as.character(dat2[[phylo_var]]))
            Csub <- phylo_cor[sp, sp, drop = FALSE]
            random_list <- c(random_list, list(stats::as.formula(paste0("~ 1 | ", phylo_var))))
            R_arg <- stats::setNames(list(Csub), phylo_var)
            used_phylo <- TRUE
        } else {
            warning(
                "Phylogenetic random effect skipped (species absent from correlation matrix): ",
                paste(missing_sp, collapse = ", ")
            )
        }
    }

    # 3. Fit Multi-Level Random-Effects Model
    mod <- metafor::rma.mv(
        yi = dat2[[yi]],
        V = VCV,
        random = random_list,
        R = R_arg,
        test = "t",
        method = "REML",
        sparse = TRUE,
        data = dat2
    )

    list(data = dat2, VCV = VCV, model = mod, used_phylo = used_phylo)
}

# Helper function to fit bias-corrected multilevel models with a continuous moderator (Egger-type)
fit_corrected_mv <- function(dat, yi = "yi", vi = "vi", moderator = "vi", cluster_var = "identifierStudyId", obs_var = "identifierEffectSizeID", rho = 0.5) {
    # Remove rows with missing values
    dat2 <- dat %>%
        filter(!is.na(.data[[yi]]), !is.na(.data[[vi]]), .data[[vi]] > 0)

    if (nrow(dat2) == 0) {
        message("No complete cases found.")
        return(NULL)
    }

    if (moderator == "se" && !"se" %in% colnames(dat2)) {
        dat2 <- dat2 %>% mutate(se = sqrt(vi))
    }

    # 1. Build VCV Matrix
    VCV <- metafor::vcalc(
        vi = dat2[[vi]],
        cluster = dat2[[cluster_var]],
        obs = dat2[[obs_var]],
        rho = rho,
        data = dat2
    )

    # 2. Fit Multi-Level Egger Model using dynamic formula
    formula_str <- paste("~ 1 +", moderator)
    mod <- metafor::rma.mv(
        yi = dat2[[yi]],
        V = VCV,
        mods = as.formula(formula_str),
        random = list(
            ~ 1 | identifierStudyId,
            ~ 1 | identifierEffectSizeID,
            ~ 1 | taxonomySpecies
        ),
        test = "t",
        method = "REML",
        sparse = TRUE,
        data = dat2
    )

    list(data = dat2, VCV = VCV, model = mod)
}

# Helper function to add Mean (95% CI) labels dynamically one line above k-values in orchard plots
add_ci_labels <- function(plot_obj, orchard_results) {
    est <- sprintf("%.2f", orchard_results$mod_table$estimate)
    ci_l <- sprintf("%.2f", orchard_results$mod_table$lowerCL)
    ci_u <- sprintf("%.2f", orchard_results$mod_table$upperCL)

    orig_labels <- plot_obj$layers[[6]]$aes_params$label

    new_labels <- c()
    for (i in 1:length(orig_labels)) {
        # Extract the original sample size numbers using regex
        nums <- regmatches(orig_labels[i], gregexpr("[0-9]+", orig_labels[i]))[[1]]
        if (length(nums) >= 2) {
            k_val <- nums[1]
            n_val <- nums[2]
            k_text <- paste0("k = ", k_val, " (", n_val, ")")
        } else {
            k_text <- orig_labels[i]
        }
        # Place Mean (95% CI) on the first line and k = xx (xx) on the second line
        new_labels[i] <- paste0(est[i], " [", ci_l[i], ", ", ci_u[i], "]\n", k_text)
    }

    plot_obj$layers[[6]]$aes_params$label <- new_labels
    plot_obj$layers[[6]]$geom_params$parse <- FALSE
    plot_obj$layers[[6]]$aes_params$hjust <- "right"
    plot_obj$layers[[6]]$aes_params$vjust <- 0.5
    plot_obj$layers[[6]]$aes_params$size <- 3.0 # Slightly smaller font size to fit better

    plot_obj
}

Combined Correction Comparison

The tabs above present the uncorrected and bias-corrected estimates on separate plots. To make the comparison easier to read, we produce two figures — one for Hedges’ \(g\) and one for the log odds ratio — using the same orchaRd::pub_bias_plot() approach as the publication-bias chapter. Each figure stacks the three model levels (New Extraction, Original, Combined) as rows; within each row a single orchard trunk shows the uncorrected multilevel mean drawn over its raw effect sizes once, with the two corrected means overlaid on top. The raw points are coloured by data source and the legend is boxed in the corner, matching the side-by-side comparison plots above.

The three estimates shown per subplot are:

  • Uncorrected (the trunk) — the multilevel rma.mv mean.
  • Bias-Corrected (Nakagawa et al. (2022)); purple overlay — the intercept of the multilevel meta-regression with sampling variance (\(v_i\)) as a moderator, evaluated at \(v_i = 0\) (the effect a hypothetical infinitely-precise study would report).
  • Bias-Robust (Yang et al. (2024)); green overlay — a fixed-/common-effect model with cluster-robust (CR2) variance estimation clustered by study.

# Yang et al. (2024) bias-robust estimate (fixed-effect + CR2 cluster-robust),
# fitted for each database split. Mirrors the construction in publication_bias.qmd
# but applied to New, Original, and Combined subsets.
fit_bias_robust <- function(dat, yi = "yi", vi = "vi", cluster_var = "identifierStudyId") {
    dat2 <- dat %>% filter(!is.na(.data[[yi]]), !is.na(.data[[vi]]), .data[[vi]] > 0)
    if (nrow(dat2) == 0) return(NULL)
    tryCatch({
        fe <- metafor::rma(yi = dat2[[yi]], vi = dat2[[vi]], method = "FE", test = "t")
        metafor::robust(fe, cluster = dat2[[cluster_var]], clubSandwich = TRUE)
    }, error = function(e) {
        warning("Bias-robust fit failed: ", conditionMessage(e)); NULL
    })
}

# Extract the corrected meta-analytic mean + 95% CI from an (intercept-only or
# vi-moderated) rma / rma.mv model — same intercept used by orchaRd::get_ints_dat.
corrected_est <- function(model) {
    b <- tryCatch(as.numeric(model$b["intrcpt", ]),
                  error = function(e) as.numeric(model$b[1]))
    data.frame(name = "Intrcpt", pred = b,
               ci.lb = model$ci.lb[1], ci.ub = model$ci.ub[1])
}

make_pub_bias_subplot <- function(unc_res, corr_res, rob_model, src, xlab, title, xlim = NULL) {

    if (length(src) == 1) src <- rep(src, nrow(h$data))
    class(h) <- "list"

    class(h) <- c("orchard", "data.frame")

# Shared effect-size range across the three panels of a stack, padded for the
# right-aligned estimate text. Includes raw points + all corrected-estimate CIs.
compute_stack_xlim <- function(db, corr_models, rob_models) {
    ci_vals <- function(m) if (is.null(m)) numeric(0) else c(m$ci.lb[1], m$ci.ub[1])
    vals <- c(db$yi,
              unlist(lapply(corr_models, function(r) if (is.null(r)) numeric(0) else ci_vals(r$model))),
              unlist(lapply(rob_models, ci_vals)))
    vals <- vals[is.finite(vals)]
    rng <- range(vals)
    pad <- diff(rng) * 0.08
    c(rng[1] - pad, rng[2] + pad)
}

Taxonomic Moderator (Heteroscedastic Model)

The combined synthesis pools effect sizes across very different organisms — insects and other arthropods, ray-finned fishes, and the remaining vertebrates (birds and mammals). Two questions follow naturally: does the average strength of mate-choice copying differ among these groups, and is the amount of heterogeneity itself group-specific? To answer both we fit a single rma.mv model with taxonomic group as a categorical moderator and a heteroscedastic random-effects structure that lets the variance components differ across groups.

Three design choices define this model:

  • Taxonomic grouping. Each species (full binomial) is assigned to Arthropods, Fish, or Other Vertebrates by genus. The genus uniquely determines the class for every species in these data, so the mapping is unambiguous (verified in the table below).
  • No phylogenetic term. The heterogeneity decomposition shows the phylogenetic component contributes very little variance, so it is dropped here; the species random intercept (taxonomySpecies) still absorbs among-species differences within each group.
  • Heteroscedasticity (struct = "HCS", rho = 0). The residual (within-study) effect-size variance is allowed to take a separate value in each taxonomic group, over a single pooled between-study variance, rather than forcing one common residual variance. This follows the orchaRd 2.0 recommendation for meta-regression with heterogeneous variances (Nakagawa et al. 2023): the heteroscedastic random term is written as an ~ moderator | unit formula (~ tax_group | identifierEffectSizeID) with a heteroscedastic compound-symmetry structure and the between-level covariance fixed to zero (rho = 0), since each effect size belongs to exactly one taxonomic group and a cross-group covariance is not identifiable. (A fuller structure that also made the between-study variance group-specific was considered but dropped: its per-group between-study variances collapse to zero for the groups with few studies, so it is over-parameterised for these data.)
  • Marginalised means. The moderator is fitted with an intercept (mods = ~ tax_group) and the per-group means, confidence intervals, and prediction intervals are obtained as marginalised means through orchaRd::mod_results() (which calls emmeans internally), again as recommended in the orchaRd 2.0 workflow. The overall test for differences among groups is the model’s omnibus moderator test (\(Q_M\)).

# ---------------------------------------------------------------------------
# Heteroscedastic taxonomic-moderator model, following the orchaRd 2.0
# heterogeneous-variance workflow (Nakagawa et al. 2023, vignette Example 4).
#
# * The moderator is fitted WITH an intercept (`mods = ~ tax_group`); per-group
#   means / CIs / PIs are obtained downstream as marginalised means via
#   orchaRd::mod_results() (emmeans), and the among-group test is the model's
#   omnibus moderator test (QM).
# * The heterogeneous (residual) variance is specified as a `~ tax_group | unit`
#   random term with struct = "HCS" and the between-level covariance fixed to
#   zero (rho = 0) — exactly the structure used in the orchaRd documentation.
#   With each unit belonging to a single group this estimates one residual
#   variance per group with no cross-group covariance.
# * PRIMARY structure: group-specific RESIDUAL variance
#   (`~ tax_group | identifierEffectSizeID`) over a single pooled between-study
#   variance (`~ 1 | identifierStudyId`) and a homoscedastic species intercept
#   (`~ 1 | taxonomySpecies`). NO phylogenetic term. (The fuller both-level
#   structure was dropped: its per-group between-study variances collapse to
#   zero for the thin groups, i.e. it is over-parameterised for these data.)
# * Each heteroscedastic structure is tried with several optimisers (nlminb,
#   optim/BFGS, optim/L-BFGS-B, then a high-iteration Nelder-Mead) before being
#   declared a failure; only if none converge does the cascade fall back to a
#   simpler structure. The vignette's default Nelder-Mead alone hits its
#   iteration cap on these data (convergence = 1), so robust optimisers go first.
# ---------------------------------------------------------------------------
fit_taxon_hetero_mv <- function(dat, group_var = "tax_group", yi = "yi", vi = "vi",
                                cluster_var = "identifierStudyId",
                                obs_var = "identifierEffectSizeID", rho = 0.5) {
    dat2 <- dat %>%
        filter(!is.na(.data[[yi]]), !is.na(.data[[vi]]), .data[[vi]] > 0, !is.na(.data[[group_var]]))
    if (nrow(dat2) == 0) {
        message("No complete cases found.")
        return(NULL)
    }
    dat2[[group_var]] <- droplevels(factor(dat2[[group_var]], levels = tax_levels))

    VCV <- metafor::vcalc(vi = dat2[[vi]], cluster = dat2[[cluster_var]], obs = dat2[[obs_var]], rho = rho, data = dat2)

    mods_f  <- stats::as.formula(paste0("~ ", group_var))                       # intercept-coded
    obs_f   <- stats::as.formula(paste0("~ ", group_var, " | ", obs_var))       # heteroscedastic residual

    # Optimisers tried (in order) for the heteroscedastic structures. The default
    # Nelder-Mead used in the orchaRd vignette hits its iteration cap on these
    # data (convergence = 1), so we lead with the more robust derivative-based
    # optimisers and only then fall back to a high-iteration Nelder-Mead.
    opt_grid <- list(
        "nlminb"            = list(optimizer = "nlminb", iter.max = 10000, eval.max = 10000),
        "optim/BFGS"        = list(optimizer = "optim", optmethod = "BFGS", maxit = 10000),
        "optim/L-BFGS-B"    = list(optimizer = "optim", optmethod = "L-BFGS-B", maxit = 10000),
        "optim/Nelder-Mead" = list(optimizer = "optim", optmethod = "Nelder-Mead", maxit = 50000, reltol = 1e-9)
    )

    # Build one rma.mv call for a candidate structure + optimiser control.
    build <- function(cand, ctrl) {
        args <- c(
            list(yi = dat2[[yi]], V = VCV, mods = mods_f, random = cand$random,
                 test = "t", dfs = "contain", method = "REML", sparse = TRUE, data = dat2),
            cand$args
        )
        if (!is.null(cand$struct)) args$struct <- cand$struct
        if (!is.null(ctrl))        args$control <- ctrl
        do.call(metafor::rma.mv, args)
    }

    # Candidate structures, tried in order. The cascade stops at the FIRST one
    # that fits without error; warnings do NOT stop the cascade but are recorded.
    # PRIMARY = residual-only HCS (orchaRd Example 4 default): a single pooled
    # between-study variance plus a group-specific residual variance. The fuller
    # both-level structure was dropped because its per-group between-study
    # variances collapse to the zero boundary for the thin groups (Other
    # Vertebrates, Arthropods) — i.e. it is over-parameterised for these data.
    candidates <- list(
        list(spec = "Heteroscedastic (residual only, per group; HCS, rho = 0)",
             random = list(~ 1 | identifierStudyId, obs_f, ~ 1 | taxonomySpecies),
             struct = "HCS", args = list(rho = 0), het = TRUE),
        list(spec = "Homoscedastic (single variance per component)",
             random = list(~ 1 | identifierStudyId, ~ 1 | identifierEffectSizeID, ~ 1 | taxonomySpecies),
             struct = NULL, args = list(), het = FALSE)
    )

    # Attempt one candidate. Heteroscedastic structures cycle through opt_grid
    # until one converges; the homoscedastic structure uses the metafor default.
    # Errors AND warnings are captured (warnings do not abort).
    fit_once <- function(cand, ctrl) {
        warns <- character(0)
        t0 <- Sys.time()
        mod <- tryCatch(
            withCallingHandlers(
                build(cand, ctrl),
                warning = function(w) {
                    warns[[length(warns) + 1L]] <<- conditionMessage(w)
                    invokeRestart("muffleWarning")
                }
            ),
            error = function(e) structure(list(msg = conditionMessage(e)), class = "tax_fit_error")
        )
        list(mod = mod, warns = warns, secs = as.numeric(difftime(Sys.time(), t0, units = "secs")))
    }

    attempt <- function(cand) {
        if (!isTRUE(cand$het)) {
            r <- fit_once(cand, NULL)
            failed <- inherits(r$mod, "tax_fit_error")
            return(list(model = if (failed) NULL else r$mod, failed = failed,
                        error = if (failed) r$mod$msg else NA_character_,
                        warnings = r$warns, secs = r$secs, optimizer = "default (nlminb)"))
        }
        msgs <- character(0)
        total <- 0
        for (onm in names(opt_grid)) {
            r <- fit_once(cand, opt_grid[[onm]])
            total <- total + r$secs
            if (!inherits(r$mod, "tax_fit_error")) {
                return(list(model = r$mod, failed = FALSE, error = NA_character_,
                            warnings = r$warns, secs = total, optimizer = onm))
            }
            msgs <- c(msgs, paste0(onm, ": ", r$mod$msg))
        }
        list(model = NULL, failed = TRUE, error = paste(msgs, collapse = " || "),
             warnings = character(0), secs = total, optimizer = NA_character_)
    }

    diag_rows <- list()
    chosen <- NULL
    chosen_spec <- NA_character_
    for (i in seq_along(candidates)) {
        cand <- candidates[[i]]
        if (!is.null(chosen)) {
            diag_rows[[i]] <- tibble::tibble(
                stage = i, structure = cand$spec, status = "not attempted", optimizer = NA_character_,
                seconds = NA_real_, n_warnings = NA_integer_, detail = "an earlier structure succeeded"
            )
            next
        }
        a <- attempt(cand)
        status <- if (a$failed) "ERROR" else if (length(a$warnings) > 0) "OK (with warnings)" else "OK"
        detail <- if (a$failed) a$error else if (length(a$warnings) > 0) paste(a$warnings, collapse = " || ") else "clean fit"
        diag_rows[[i]] <- tibble::tibble(
            stage = i, structure = cand$spec, status = status, optimizer = a$optimizer,
            seconds = round(a$secs, 2), n_warnings = length(a$warnings), detail = detail
        )
        if (!a$failed) {
            chosen <- a$model
            chosen_spec <- cand$spec
        }
    }
    diagnostics <- dplyr::bind_rows(diag_rows)

    if (is.null(chosen)) {
        warning("All taxonomic-moderator structures failed to fit.")
        return(list(data = dat2, VCV = VCV, model = NULL, spec = "ALL FAILED", diagnostics = diagnostics))
    }
    list(data = dat2, VCV = VCV, model = chosen, spec = chosen_spec, diagnostics = diagnostics)
}

# Total I^2 (orchaRd if available; otherwise a Higgins typical-variance fallback
# so a heterogeneity figure is always reported even under the heteroscedastic
# structure).
total_i2 <- function(res) {
    if (is.null(res) || is.null(res$model)) {
        return(NA_real_)
    }
    m <- res$model
    i2 <- tryCatch(orchaRd::i2_ml(m), error = function(e) NULL)
    if (!is.null(i2) && any(grepl("Total", names(i2)))) {
        return(as.numeric(i2[grep("Total", names(i2))[1]]))
    }
    vi <- res$data$vi
    wi <- 1 / vi
    k  <- length(vi)
    s2 <- (k - 1) * sum(wi) / (sum(wi)^2 - sum(wi^2))
    vc <- sum(c(m$sigma2, m$tau2, m$gamma2), na.rm = TRUE)
    100 * vc / (vc + s2)
}

Heterogeneity Decomposition (Partial \(I^2\))

Total heterogeneity (\(I^2_{\text{Total}}\)) quantifies the proportion of variance in the observed effect sizes that reflects true differences among effects rather than sampling error. In a multilevel model this total can be partitioned into the contribution of each random effect (Nakagawa et al. 2022). Here the variance is split among four components: between-study differences, the residual effect-size (within-study) level, a species (non-phylogenetic) component, and a phylogenetic component that captures variance shared among species in proportion to their relatedness. We compute these partial \(I^2\) values with orchaRd::i2_ml() for each uncorrected model.

Interpreting the Heterogeneity

The combined models show that mate-choice copying effect sizes are highly heterogeneous: total \(I^2\) reaches 85.7% for Hedges’ \(g\) and 74.7% for the log odds ratio. In other words, the large majority of the observed variation reflects genuine differences among effects rather than sampling error alone, so the pooled means should be read as averages over a broad distribution of true effects rather than as a single common effect.

Decomposing that heterogeneity is where the phylogeny becomes informative. For the combined Hedges’ \(g\) model, the phylogenetic component accounts for 3.5% of the variance, the non-phylogenetic species component for 3.1%, between-study differences for 7.4%, and the residual effect-size level for 71.7%. The combined log-odds-ratio model partitions similarly: phylogeny 1.1%, species 23.9%, study 4.3%, and residual 45.4%.

The size of the phylogenetic component is the key new result. A substantial value would indicate that closely related species tend to show similar degrees of mate-choice copying — i.e., the trait carries a phylogenetic signal and effects are not independent across the tree — which would caution against treating species as exchangeable and would motivate phylogenetically informed interpretation. A small phylogenetic component, by contrast, indicates that relatedness explains little of the among-species variation: most of the species-level differences are idiosyncratic (captured by the non-phylogenetic species term) rather than inherited, and the earlier non-phylogenetic models were not materially biased by ignoring the tree. The contrast between the phylogenetic and non-phylogenetic species components is therefore the most direct readout of how much shared ancestry, as opposed to species-specific biology, structures the data.

Two further patterns are worth noting. First, the residual effect-size level typically absorbs a large share of the variance, reflecting heterogeneity among individual trials within studies (different paradigms, demonstrators, and conditions). Second, comparing the New-Only, Original-Only, and Combined columns shows whether the new extraction introduces or dilutes heterogeneity relative to the legacy datasets; where the new and original subsets allocate variance differently, the combined estimate reflects a weighted blend of the two. Because this high heterogeneity is the rule rather than the exception, it justifies the moderator and sub-group analyses pursued elsewhere in the workflow, and it is the reason the pooled estimates are reported with prediction intervals in addition to confidence intervals.

Sensitivity Analysis: Impact of the VCV Structure

To evaluate the robustness of our meta-analytic results, we conduct a sensitivity analysis comparing the main multilevel models (which incorporate a Variance-Covariance [VCV] matrix with an assumed within-study correlation \(\rho = 0.5\)) against simplified multilevel models that ignore this correlation (i.e., treating effect size variances as independent diagonal elements, \(V = v_i\)).

Ignoring within-study correlation can lead to improper weighting of studies with multiple effect sizes and can bias the standard errors and confidence intervals of the pooled estimates.

# 1. Define comparison function
compare_vcv_models <- function(dat, name, yi = "yi", vi = "vi", cluster_var = "identifierStudyId", obs_var = "identifierEffectSizeID", rho = 0.5) {
    dat2 <- dat %>%
        filter(!is.na(.data[[yi]]), !is.na(.data[[vi]]), .data[[vi]] > 0)

    if (nrow(dat2) == 0) {
        return(NULL)
    }

    # A. With VCV (rho = 0.5)
    VCV <- metafor::vcalc(
        vi = dat2[[vi]],
        cluster = dat2[[cluster_var]],
        obs = dat2[[obs_var]],
        rho = rho,
        data = dat2
    )
    mod_w_vcv <- metafor::rma.mv(
        yi = dat2[[yi]],
        V = VCV,
        random = list(
            ~ 1 | identifierStudyId,
            ~ 1 | identifierEffectSizeID,
            ~ 1 | taxonomySpecies
        ),
        test = "t",
        method = "REML",
        sparse = TRUE,
        data = dat2
    )

    # B. Without VCV (diagonal V = vi)
    mod_no_vcv <- metafor::rma.mv(
        yi = dat2[[yi]],
        V = dat2[[vi]],
        random = list(
            ~ 1 | identifierStudyId,
            ~ 1 | identifierEffectSizeID,
            ~ 1 | taxonomySpecies
        ),
        test = "t",
        method = "REML",
        sparse = TRUE,
        data = dat2
    )

    # Combine results
    data.frame(
        Analysis = name,
        Model_Type = c("With VCV (rho = 0.5)", "Without VCV (Independent)"),
        k = c(mod_w_vcv$k, mod_no_vcv$k),
        Estimate = c(mod_w_vcv$b[1], mod_no_vcv$b[1]),
        SE = c(mod_w_vcv$se[1], mod_no_vcv$se[1]),
        CI_Lower = c(mod_w_vcv$ci.lb[1], mod_no_vcv$ci.lb[1]),
        CI_Upper = c(mod_w_vcv$ci.ub[1], mod_no_vcv$ci.ub[1]),
        p_val = c(mod_w_vcv$pval[1], mod_no_vcv$pval[1])
    )
}

Key Methodological Findings

  1. Marginal Shifts in Pooled Estimates: Treating effect sizes as completely independent (Without VCV) causes very minor changes to the overall pooled estimates.   2. Standard Error Shifts: In this multilevel model specification (which partitions variance using nested random intercepts), omitting the VCV matrix and assuming independence leads to a slight inflation of the standard error and wider confidence intervals for the combined analyses:

    • Hedges’ \(g\) (Combined): SE increases from \(0.07\) to \(0.07\) (CI width increases from \(0.29\) to \(0.30\)).
    • Log Odds Ratio (Combined): SE increases from \(0.14\) to \(0.15\) (CI width increases from \(0.54\) to \(0.59\)).

  3. Impact on Statistical Significance: For continuous outcomes (Hedges’ \(g\) Combined), the pooled estimate is statistically significant with the VCV matrix (\(p < 0.0001\)) and statistically significant without it (\(p < 0.0001\)). Binary outcomes (Log OR Combined) are statistically significant under both specifications (\(p < 0.0001\) with VCV; \(p < 0.0001\) without).

* **New Extractions Only:** Eduardo's new dataset.

## Model Fitting Function

This helper function handles the variance–covariance (VCV) matrix construction and fits a multilevel random-effects model using `rma.mv()` in `metafor`. 

* **Variance–Covariance:** We construct a VCV matrix via `vcalc()` to handle dependencies among multiple effect sizes within the same study (assuming $r = 0.5$ for nested measurements).
* **Random intercepts** are modeled for:

  * Study (`identifierStudyId`)
  * Individual trial/effect size (`identifierEffectSizeID`)
  * Species (`taxonomySpecies`) — the *non-phylogenetic* species effect, capturing species-specific deviations not explained by shared ancestry
  * Phylogeny (`species_phylo`) — a *phylogenetic* random effect supplied with a species-by-species correlation matrix (`R = list(species_phylo = phylo_cor)`) derived from the Open Tree of Life topology with Grafen branch lengths (see `scripts/2_phylogeny/build_phylogeny.R`). This partitions the among-species variance into a component attributable to phylogenetic relatedness and a residual species component.

The phylogenetic term is added to the **uncorrected** models only (for the New, Original, and Combined datasets); the small-study bias-corrected (Egger-type) models retain the original structure. When a phylogenetic correlation matrix is unavailable, the fit gracefully falls back to the non-phylogenetic structure.

```{r}
#| label: model_fitting_function
fit_overall_mv <- function(dat, yi = "yi", vi = "vi", cluster_var = "identifierStudyId", obs_var = "identifierEffectSizeID", rho = 0.5, phylo_cor = NULL, phylo_var = "species_phylo") {
    # Remove rows with missing values
    dat2 <- dat %>%
        filter(!is.na(.data[[yi]]), !is.na(.data[[vi]]), .data[[vi]] > 0)

    if (nrow(dat2) == 0) {
        message("No complete cases found.")
        return(NULL)
    }

    # 1. Build VCV Matrix
    VCV <- metafor::vcalc(
        vi = dat2[[vi]],
        cluster = dat2[[cluster_var]],
        obs = dat2[[obs_var]],
        rho = rho,
        data = dat2
    )

    # 2. Assemble random-effects structure. Always include study, effect-size,
    #    and a (non-phylogenetic) species intercept. If a phylogenetic
    #    correlation matrix is supplied AND every species level is present in it,
    #    add a phylogenetic random effect with R = subset-of-correlation-matrix.
    random_list <- list(
        ~ 1 | identifierStudyId,
        ~ 1 | identifierEffectSizeID,
        ~ 1 | taxonomySpecies
    )
    R_arg <- NULL
    used_phylo <- FALSE

    if (!is.null(phylo_cor) && phylo_var %in% colnames(dat2)) {
        sp <- unique(as.character(dat2[[phylo_var]]))
        missing_sp <- setdiff(sp, rownames(phylo_cor))
        if (length(missing_sp) == 0) {
            # Subset the correlation matrix to the species actually present so the
            # dimnames match the grouping levels exactly. Also drop any stale
            # (unused) factor levels on the grouping column so metafor's level
            # set matches the R-matrix dimnames.
            dat2[[phylo_var]] <- factor(as.character(dat2[[phylo_var]]))
            Csub <- phylo_cor[sp, sp, drop = FALSE]
            random_list <- c(random_list, list(stats::as.formula(paste0("~ 1 | ", phylo_var))))
            R_arg <- stats::setNames(list(Csub), phylo_var)
            used_phylo <- TRUE
        } else {
            warning(
                "Phylogenetic random effect skipped (species absent from correlation matrix): ",
                paste(missing_sp, collapse = ", ")
            )
        }
    }

    # 3. Fit Multi-Level Random-Effects Model
    mod <- metafor::rma.mv(
        yi = dat2[[yi]],
        V = VCV,
        random = random_list,
        R = R_arg,
        test = "t",
        method = "REML",
        sparse = TRUE,
        data = dat2
    )

    list(data = dat2, VCV = VCV, model = mod, used_phylo = used_phylo)
}

# Helper function to fit bias-corrected multilevel models with a continuous moderator (Egger-type)
fit_corrected_mv <- function(dat, yi = "yi", vi = "vi", moderator = "vi", cluster_var = "identifierStudyId", obs_var = "identifierEffectSizeID", rho = 0.5) {
    # Remove rows with missing values
    dat2 <- dat %>%
        filter(!is.na(.data[[yi]]), !is.na(.data[[vi]]), .data[[vi]] > 0)

    if (nrow(dat2) == 0) {
        message("No complete cases found.")
        return(NULL)
    }

    if (moderator == "se" && !"se" %in% colnames(dat2)) {
        dat2 <- dat2 %>% mutate(se = sqrt(vi))
    }

    # 1. Build VCV Matrix
    VCV <- metafor::vcalc(
        vi = dat2[[vi]],
        cluster = dat2[[cluster_var]],
        obs = dat2[[obs_var]],
        rho = rho,
        data = dat2
    )

    # 2. Fit Multi-Level Egger Model using dynamic formula
    formula_str <- paste("~ 1 +", moderator)
    mod <- metafor::rma.mv(
        yi = dat2[[yi]],
        V = VCV,
        mods = as.formula(formula_str),
        random = list(
            ~ 1 | identifierStudyId,
            ~ 1 | identifierEffectSizeID,
            ~ 1 | taxonomySpecies
        ),
        test = "t",
        method = "REML",
        sparse = TRUE,
        data = dat2
    )

    list(data = dat2, VCV = VCV, model = mod)
}

# Helper function to add Mean (95% CI) labels dynamically one line above k-values in orchard plots
add_ci_labels <- function(plot_obj, orchard_results) {
    est <- sprintf("%.2f", orchard_results$mod_table$estimate)
    ci_l <- sprintf("%.2f", orchard_results$mod_table$lowerCL)
    ci_u <- sprintf("%.2f", orchard_results$mod_table$upperCL)

    orig_labels <- plot_obj$layers[[6]]$aes_params$label

    new_labels <- c()
    for (i in 1:length(orig_labels)) {
        # Extract the original sample size numbers using regex
        nums <- regmatches(orig_labels[i], gregexpr("[0-9]+", orig_labels[i]))[[1]]
        if (length(nums) >= 2) {
            k_val <- nums[1]
            n_val <- nums[2]
            k_text <- paste0("k = ", k_val, " (", n_val, ")")
        } else {
            k_text <- orig_labels[i]
        }
        # Place Mean (95% CI) on the first line and k = xx (xx) on the second line
        new_labels[i] <- paste0(est[i], " [", ci_l[i], ", ", ci_u[i], "]\n", k_text)
    }

    plot_obj$layers[[6]]$aes_params$label <- new_labels
    plot_obj$layers[[6]]$geom_params$parse <- FALSE
    plot_obj$layers[[6]]$aes_params$hjust <- "right"
    plot_obj$layers[[6]]$aes_params$vjust <- 0.5
    plot_obj$layers[[6]]$aes_params$size <- 3.0 # Slightly smaller font size to fit better

    plot_obj
}
```

## Combined Correction Comparison {#sec-correction-comparison}

The tabs above present the uncorrected and bias-corrected estimates on separate plots. To make the comparison easier to read, we produce **two figures** — one for Hedges' $g$ and one for the log odds ratio — using the same `orchaRd::pub_bias_plot()` approach as the publication-bias chapter. Each figure stacks the three model levels (New Extraction, Original, Combined) as rows; within each row a single orchard trunk shows the uncorrected multilevel mean drawn over its raw effect sizes **once**, with the two corrected means overlaid on top. The raw points are coloured by data source and the legend is boxed in the corner, matching the side-by-side comparison plots above.

The three estimates shown per subplot are:

* **Uncorrected** (the trunk) — the multilevel `rma.mv` mean.
* **Bias-Corrected (Nakagawa et al. [-@nakagawaMethodsTestingPublication2022]); purple overlay** — the intercept of the multilevel meta-regression with sampling variance ($v_i$) as a moderator, evaluated at $v_i = 0$ (the effect a hypothetical infinitely-precise study would report).
* **Bias-Robust (Yang et al. [-@yangRobustPointVariance2024]); green overlay** — a fixed-/common-effect model with cluster-robust (CR2) variance estimation clustered by study.

```{r}
#| label: fit_bias_robust_splits
#| message: false
#| warning: false
# Yang et al. (2024) bias-robust estimate (fixed-effect + CR2 cluster-robust),
# fitted for each database split. Mirrors the construction in publication_bias.qmd
# but applied to New, Original, and Combined subsets.
fit_bias_robust <- function(dat, yi = "yi", vi = "vi", cluster_var = "identifierStudyId") {
    dat2 <- dat %>% filter(!is.na(.data[[yi]]), !is.na(.data[[vi]]), .data[[vi]] > 0)
    if (nrow(dat2) == 0) return(NULL)
    tryCatch({
        fe <- metafor::rma(yi = dat2[[yi]], vi = dat2[[vi]], method = "FE", test = "t")
        metafor::robust(fe, cluster = dat2[[cluster_var]], clubSandwich = TRUE)
    }, error = function(e) {
        warning("Bias-robust fit failed: ", conditionMessage(e)); NULL
    })
}

rob_hedges_orig <- fit_bias_robust(db_hedges %>% filter(Dataset_Source == "Original (Davies 2020)"))
rob_hedges_comb <- fit_bias_robust(db_hedges)
```

```{r}
# Extract the corrected meta-analytic mean + 95% CI from an (intercept-only or
# vi-moderated) rma / rma.mv model — same intercept used by orchaRd::get_ints_dat.
corrected_est <- function(model) {
    b <- tryCatch(as.numeric(model$b["intrcpt", ]),
                  error = function(e) as.numeric(model$b[1]))
    data.frame(name = "Intrcpt", pred = b,
               ci.lb = model$ci.lb[1], ci.ub = model$ci.ub[1])
}

# src: either a single data-source label (recycled) or a per-point character
# vector matching the order of the model's effect sizes (used for Combined).
# xlim: optional c(lo, hi) shared effect-size range for axis alignment; when
# supplied, the estimate text is right-aligned to xlim[2] (inside the box).
make_pub_bias_subplot <- function(unc_res, corr_res, rob_model, src, xlab, title, xlim = NULL) {
    if (is.null(unc_res) || is.null(unc_res$model) ||
        is.null(corr_res) || is.null(corr_res$model) || is.null(rob_model)) {
        return(NULL)
    }
    # Single-trunk orchard result; attach the true per-point data source.
    h <- orchaRd::mod_results(unc_res$model, group = "identifierStudyId")
    if (length(src) == 1) src <- rep(src, nrow(h$data))
    class(h) <- "list"
    h$data$real_source <- src
    class(h) <- c("orchard", "data.frame")

    # k/g labels to the left so the right margin is free for the estimate text.
    # Widen point.size range so bubble area tracks precision more visibly.
    # trunk.size = 1 -> point estimate drawn at size 4 (matches the overlay diamonds).
    oc <- orchaRd::orchard_plot(h, xlab = xlab, flip = TRUE, k.pos = "left",
                                point.size = c(0.8, 10), trunk.size = 1)

    # Recolour the raw points (layer 1) by data source — as in hedges_comparison_plot.
    oc$layers[[1]]$mapping$colour <- ggplot2::aes(colour = real_source)$colour
    oc$layers[[1]]$mapping$fill   <- ggplot2::aes(fill = real_source)$fill

    # Corrected estimates to overlay on the single uncorrected trunk.
    yang <- corrected_est(rob_model)       # Bias-Robust   (Yang 2024)
    naka <- corrected_est(corr_res$model)  # Bias-Corrected (Nakagawa 2022)

    # Common effect-size range (falls back to this panel's data if not supplied).
    if (is.null(xlim)) {
        rng <- range(c(h$data$yi, yang$ci.lb, yang$ci.ub, naka$ci.lb, naka$ci.ub),
                     na.rm = TRUE, finite = TRUE)
        pad <- diff(rng) * 0.08
        xlim <- c(rng[1] - pad, rng[2] + pad)
    }
    txt_y <- xlim[2]  # right edge (effect-size axis is horizontal after flip)

    oc <- oc +
        # Bias-Robust (Yang) — nudged below the trunk. Black CI bar; coloured
        # diamond with a black border so it stands out against the bubbles.
        ggplot2::geom_linerange(data = yang,
            ggplot2::aes(x = name, ymin = ci.lb, ymax = ci.ub),
            colour = "black", linewidth = 1.2,
            position = ggplot2::position_nudge(x = -0.08), inherit.aes = FALSE) +
        ggplot2::geom_point(data = yang,
            ggplot2::aes(x = name, y = pred),
            fill = yang_col, colour = "black", shape = 23, size = 4, stroke = 0.9,
            position = ggplot2::position_nudge(x = -0.08), inherit.aes = FALSE) +
        # Bias-Corrected (Nakagawa) — nudged above the trunk. Black CI bar;
        # coloured diamond with a black border.
        ggplot2::geom_linerange(data = naka,
            ggplot2::aes(x = name, ymin = ci.lb, ymax = ci.ub),
            colour = "black", linewidth = 1.2,
            position = ggplot2::position_nudge(x = 0.08), inherit.aes = FALSE) +
        ggplot2::geom_point(data = naka,
            ggplot2::aes(x = name, y = pred),
            fill = naka_col, colour = "black", shape = 23, size = 4, stroke = 0.9,
            position = ggplot2::position_nudge(x = 0.08), inherit.aes = FALSE) +
        # Estimate text, right-aligned to the right edge so it stays in the box.
        # x positions mirror each diamond's nudge: Yang below (x<1), Naka above (x>1).
        ggplot2::annotate("text", x = 0.66, y = txt_y, hjust = 1, vjust = 0.5,
            colour = yang_col, size = 3.4, fontface = "bold",
            label = sprintf("Bias-Robust (Yang): %.2f [%.2f, %.2f]",
                            yang$pred, yang$ci.lb, yang$ci.ub)) +
        ggplot2::annotate("text", x = 1.34, y = txt_y, hjust = 1, vjust = 0.5,
            colour = naka_col, size = 3.4, fontface = "bold",
            label = sprintf("Bias-Corrected (Nakagawa): %.2f [%.2f, %.2f]",
                            naka$pred, naka$ci.lb, naka$ci.ub)) +
        ggplot2::scale_colour_manual(values = source_palette, name = "Data Source") +
        ggplot2::scale_fill_manual(values = source_palette, name = "Data Source") +
        ggplot2::ggtitle(title) +
        ggplot2::coord_flip(ylim = xlim) +
        ggplot2::theme(
            legend.position      = c(0.98, 0.02),
            legend.justification = c("right", "bottom"),
            legend.background    = ggplot2::element_rect(fill = scales::alpha("white", 0.75), color = "grey90"),
            legend.box           = "vertical",
            legend.margin        = ggplot2::margin(t = 4, r = 4, b = 4, l = 4),
            legend.title         = ggplot2::element_text(size = 9, face = "bold"),
            legend.text          = ggplot2::element_text(size = 8),

# Shared effect-size range across the three panels of a stack, padded for the
```

## Taxonomic Moderator (Heteroscedastic Model) {#sec-taxonomic-moderator}

The combined synthesis pools effect sizes across very different organisms — insects and other arthropods, ray-finned fishes, and the remaining vertebrates (birds and mammals). Two questions follow naturally: does the *average* strength of mate-choice copying differ among these groups, and is the *amount of heterogeneity* itself group-specific? To answer both we fit a single `rma.mv` model with **taxonomic group as a categorical moderator** and a **heteroscedastic** random-effects structure that lets the variance components differ across groups.

Three design choices define this model:

* **Taxonomic grouping.** Each species (full binomial) is assigned to **Arthropods**, **Fish**, or **Other Vertebrates** by genus. The genus uniquely determines the class for every species in these data, so the mapping is unambiguous (verified in the table below).
* **No phylogenetic term.** The heterogeneity decomposition shows the phylogenetic component contributes very little variance, so it is dropped here; the species random intercept (`taxonomySpecies`) still absorbs among-species differences within each group.
* **Heteroscedasticity (`struct = "HCS"`, `rho = 0`).** The **residual (within-study) effect-size** variance is allowed to take a separate value in each taxonomic group, over a single pooled between-study variance, rather than forcing one common residual variance. This follows the `orchaRd` 2.0 recommendation for meta-regression with heterogeneous variances [@nakagawaOrchaRd2Visualising2023]: the heteroscedastic random term is written as an `~ moderator | unit` formula (`~ tax_group | identifierEffectSizeID`) with a heteroscedastic compound-symmetry structure and the between-level covariance fixed to zero (`rho = 0`), since each effect size belongs to exactly one taxonomic group and a cross-group covariance is not identifiable. (A fuller structure that *also* made the between-study variance group-specific was considered but dropped: its per-group between-study variances collapse to zero for the groups with few studies, so it is over-parameterised for these data.)
* **Marginalised means.** The moderator is fitted **with an intercept** (`mods = ~ tax_group`) and the per-group means, confidence intervals, and prediction intervals are obtained as **marginalised means** through `orchaRd::mod_results()` (which calls `emmeans` internally), again as recommended in the `orchaRd` 2.0 workflow. The overall test for differences among groups is the model's omnibus moderator test ($Q_M$).

### Model Fitting

```{r}
#| label: fit_taxon_hetero_models
#| message: false
#| warning: false
# ---------------------------------------------------------------------------
# Heteroscedastic taxonomic-moderator model, following the orchaRd 2.0
# heterogeneous-variance workflow (Nakagawa et al. 2023, vignette Example 4).
#
# * The moderator is fitted WITH an intercept (`mods = ~ tax_group`); per-group
#   means / CIs / PIs are obtained downstream as marginalised means via
#   orchaRd::mod_results() (emmeans), and the among-group test is the model's
#   omnibus moderator test (QM).
# * The heterogeneous (residual) variance is specified as a `~ tax_group | unit`
#   random term with struct = "HCS" and the between-level covariance fixed to
#   zero (rho = 0) — exactly the structure used in the orchaRd documentation.
#   With each unit belonging to a single group this estimates one residual
#   variance per group with no cross-group covariance.
# * PRIMARY structure: group-specific RESIDUAL variance
#   (`~ tax_group | identifierEffectSizeID`) over a single pooled between-study
#   variance (`~ 1 | identifierStudyId`) and a homoscedastic species intercept
#   (`~ 1 | taxonomySpecies`). NO phylogenetic term. (The fuller both-level
#   structure was dropped: its per-group between-study variances collapse to
#   zero for the thin groups, i.e. it is over-parameterised for these data.)
# * Each heteroscedastic structure is tried with several optimisers (nlminb,
#   optim/BFGS, optim/L-BFGS-B, then a high-iteration Nelder-Mead) before being
#   declared a failure; only if none converge does the cascade fall back to a
#   simpler structure. The vignette's default Nelder-Mead alone hits its
#   iteration cap on these data (convergence = 1), so robust optimisers go first.
# ---------------------------------------------------------------------------
fit_taxon_hetero_mv <- function(dat, group_var = "tax_group", yi = "yi", vi = "vi",
                                cluster_var = "identifierStudyId",
                                obs_var = "identifierEffectSizeID", rho = 0.5) {
    dat2 <- dat %>%
        filter(!is.na(.data[[yi]]), !is.na(.data[[vi]]), .data[[vi]] > 0, !is.na(.data[[group_var]]))
    if (nrow(dat2) == 0) {
        message("No complete cases found.")
        return(NULL)
    }
    dat2[[group_var]] <- droplevels(factor(dat2[[group_var]], levels = tax_levels))

    VCV <- metafor::vcalc(vi = dat2[[vi]], cluster = dat2[[cluster_var]], obs = dat2[[obs_var]], rho = rho, data = dat2)

    mods_f  <- stats::as.formula(paste0("~ ", group_var))                       # intercept-coded
    obs_f   <- stats::as.formula(paste0("~ ", group_var, " | ", obs_var))       # heteroscedastic residual

    # Optimisers tried (in order) for the heteroscedastic structures. The default
    # Nelder-Mead used in the orchaRd vignette hits its iteration cap on these
    # data (convergence = 1), so we lead with the more robust derivative-based
    # optimisers and only then fall back to a high-iteration Nelder-Mead.
    opt_grid <- list(
        "nlminb"            = list(optimizer = "nlminb", iter.max = 10000, eval.max = 10000),
        "optim/BFGS"        = list(optimizer = "optim", optmethod = "BFGS", maxit = 10000),
        "optim/L-BFGS-B"    = list(optimizer = "optim", optmethod = "L-BFGS-B", maxit = 10000),
        "optim/Nelder-Mead" = list(optimizer = "optim", optmethod = "Nelder-Mead", maxit = 50000, reltol = 1e-9)
    )

    # Build one rma.mv call for a candidate structure + optimiser control.
    build <- function(cand, ctrl) {
        args <- c(
            list(yi = dat2[[yi]], V = VCV, mods = mods_f, random = cand$random,
                 test = "t", dfs = "contain", method = "REML", sparse = TRUE, data = dat2),
            cand$args
        )
        if (!is.null(cand$struct)) args$struct <- cand$struct
        if (!is.null(ctrl))        args$control <- ctrl
        do.call(metafor::rma.mv, args)
    }

    # Candidate structures, tried in order. The cascade stops at the FIRST one
    # that fits without error; warnings do NOT stop the cascade but are recorded.
    # PRIMARY = residual-only HCS (orchaRd Example 4 default): a single pooled
    # between-study variance plus a group-specific residual variance. The fuller
    # both-level structure was dropped because its per-group between-study
    # variances collapse to the zero boundary for the thin groups (Other
    # Vertebrates, Arthropods) — i.e. it is over-parameterised for these data.
    candidates <- list(
        list(spec = "Heteroscedastic (residual only, per group; HCS, rho = 0)",
             random = list(~ 1 | identifierStudyId, obs_f, ~ 1 | taxonomySpecies),
             struct = "HCS", args = list(rho = 0), het = TRUE),
        list(spec = "Homoscedastic (single variance per component)",
             random = list(~ 1 | identifierStudyId, ~ 1 | identifierEffectSizeID, ~ 1 | taxonomySpecies),
             struct = NULL, args = list(), het = FALSE)
    )

    # Attempt one candidate. Heteroscedastic structures cycle through opt_grid
    # until one converges; the homoscedastic structure uses the metafor default.
    # Errors AND warnings are captured (warnings do not abort).
    fit_once <- function(cand, ctrl) {
        warns <- character(0)
        t0 <- Sys.time()
        mod <- tryCatch(
            withCallingHandlers(
                build(cand, ctrl),
                warning = function(w) {
                    warns[[length(warns) + 1L]] <<- conditionMessage(w)
                    invokeRestart("muffleWarning")
                }
            ),
            error = function(e) structure(list(msg = conditionMessage(e)), class = "tax_fit_error")
        )
        list(mod = mod, warns = warns, secs = as.numeric(difftime(Sys.time(), t0, units = "secs")))
    }

    attempt <- function(cand) {
        if (!isTRUE(cand$het)) {
            r <- fit_once(cand, NULL)
            failed <- inherits(r$mod, "tax_fit_error")
            return(list(model = if (failed) NULL else r$mod, failed = failed,
                        error = if (failed) r$mod$msg else NA_character_,
                        warnings = r$warns, secs = r$secs, optimizer = "default (nlminb)"))
        }
        msgs <- character(0)
        total <- 0
        for (onm in names(opt_grid)) {
            r <- fit_once(cand, opt_grid[[onm]])
            total <- total + r$secs
            if (!inherits(r$mod, "tax_fit_error")) {
                return(list(model = r$mod, failed = FALSE, error = NA_character_,
                            warnings = r$warns, secs = total, optimizer = onm))
            }
            msgs <- c(msgs, paste0(onm, ": ", r$mod$msg))
        }
        list(model = NULL, failed = TRUE, error = paste(msgs, collapse = " || "),
             warnings = character(0), secs = total, optimizer = NA_character_)
    }

    diag_rows <- list()
    chosen <- NULL
    chosen_spec <- NA_character_
    for (i in seq_along(candidates)) {
        cand <- candidates[[i]]
        if (!is.null(chosen)) {
            diag_rows[[i]] <- tibble::tibble(
                stage = i, structure = cand$spec, status = "not attempted", optimizer = NA_character_,
                seconds = NA_real_, n_warnings = NA_integer_, detail = "an earlier structure succeeded"
            )
            next
        }
        a <- attempt(cand)
        status <- if (a$failed) "ERROR" else if (length(a$warnings) > 0) "OK (with warnings)" else "OK"
        detail <- if (a$failed) a$error else if (length(a$warnings) > 0) paste(a$warnings, collapse = " || ") else "clean fit"
        diag_rows[[i]] <- tibble::tibble(
            stage = i, structure = cand$spec, status = status, optimizer = a$optimizer,
            seconds = round(a$secs, 2), n_warnings = length(a$warnings), detail = detail
        )
        if (!a$failed) {
            chosen <- a$model
            chosen_spec <- cand$spec
        }
    }
    diagnostics <- dplyr::bind_rows(diag_rows)

    if (is.null(chosen)) {
        warning("All taxonomic-moderator structures failed to fit.")
        return(list(data = dat2, VCV = VCV, model = NULL, spec = "ALL FAILED", diagnostics = diagnostics))
    }
    list(data = dat2, VCV = VCV, model = chosen, spec = chosen_spec, diagnostics = diagnostics)
}

res_tax_hedges <- fit_taxon_hetero_mv(db_hedges_tax)
res_tax_or     <- fit_taxon_hetero_mv(db_or_tax)

# Total I^2 (orchaRd if available; otherwise a Higgins typical-variance fallback
# so a heterogeneity figure is always reported even under the heteroscedastic
# structure).
total_i2 <- function(res) {
    if (is.null(res) || is.null(res$model)) {
        return(NA_real_)
    }
    m <- res$model
    i2 <- tryCatch(orchaRd::i2_ml(m), error = function(e) NULL)
    if (!is.null(i2) && any(grepl("Total", names(i2)))) {
        return(as.numeric(i2[grep("Total", names(i2))[1]]))
    }
    vi <- res$data$vi
    wi <- 1 / vi
    k  <- length(vi)
    s2 <- (k - 1) * sum(wi) / (sum(wi)^2 - sum(wi^2))
    vc <- sum(c(m$sigma2, m$tau2, m$gamma2), na.rm = TRUE)
    100 * vc / (vc + s2)
}
```

### Fit Diagnostics {#sec-taxon-fit-diagnostics}

The model is fitted through a **fallback cascade**: the most flexible heteroscedastic structure is attempted first, and only if it *errors* does the next-simpler structure get tried (warnings do **not** trigger a fallback, but are captured and shown). The report below tells you exactly what happened for each effect-size class — which structures were attempted, whether each fit was clean / produced warnings / errored (with the captured message), how long it took, which structure was ultimately used, the per-group sample sizes, and the variance components of the chosen model (so you can see whether any group's variance collapsed to the zero boundary).

```{r}
# Variance components of the chosen model, labelled by random-effect term.
# Labelling is STRUCTURE-AWARE (keyed on res$spec) because which variance lands
# in metafor's sigma2 / tau2 / gamma2 slots depends on which cascade structure
# was used:
#   * both-level het  -> tau2 = between-study (per group), gamma2 = residual
#                        (per group), sigma2 = species
#   * residual-only   -> tau2 = residual (per group), sigma2 = c(study, species)
#   * homoscedastic   -> sigma2 = c(study, effect-size residual, species)
chosen_varcomp <- function(res) {
    if (is.null(res) || is.null(res$model)) {
        return(NULL)
    }
    m   <- res$model
    glv <- levels(droplevels(factor(res$data$tax_group, levels = tax_levels)))
    out <- tibble::tibble(Component = character(), Variance = numeric())
    addc <- function(out, labs, vals) {
        if (is.null(vals) || length(vals) == 0) {
            return(out)
        }
        dplyr::bind_rows(out, tibble::tibble(Component = labs, Variance = round(as.numeric(vals), 5)))
    }
    spec <- if (is.null(res$spec)) "" else res$spec
    if (grepl("between-study \\+ residual", spec)) {
        out <- addc(out, paste0("Between-study τ² — ", glv[seq_along(m$tau2)]), m$tau2)
        out <- addc(out, paste0("Residual σ² — ", glv[seq_along(m$gamma2)]), m$gamma2)
        out <- addc(out, "Species σ²", m$sigma2)
    } else if (grepl("residual only", spec)) {
        out <- addc(out, paste0("Residual σ² — ", glv[seq_along(m$tau2)]), m$tau2)
        out <- addc(out, c("Between-study τ²", "Species σ²")[seq_along(m$sigma2)], m$sigma2)
    } else {
        out <- addc(out, c("Between-study τ²", "Residual σ² (effect size)", "Species σ²")[seq_along(m$sigma2)], m$sigma2)
    }
    out
}
```

## Heterogeneity Decomposition (Partial $I^2$) {#sec-heterogeneity}

Total heterogeneity ($I^2_{\text{Total}}$) quantifies the proportion of variance in the observed effect sizes that reflects true differences among effects rather than sampling error. In a multilevel model this total can be *partitioned* into the contribution of each random effect [@nakagawaMethodsTestingPublication2022]. Here the variance is split among four components: between-**study** differences, the residual **effect-size** (within-study) level, a **species** (non-phylogenetic) component, and a **phylogenetic** component that captures variance shared among species in proportion to their relatedness. We compute these partial $I^2$ values with `orchaRd::i2_ml()` for each uncorrected model.

```{r}

# Pull the partial I^2 vector from a fitted model into tidy long form
get_i2 <- function(res, effect, model) {
    if (is.null(res) || is.null(res$model)) {

    }
    v <- tryCatch(orchaRd::i2_ml(res$model), error = function(e) NULL)
    if (is.null(v)) {
        return(NULL)
    }
    tibble::tibble(
        Effect    = effect,
        Model     = model,
        Component = sub("^I2_", "", names(v)),
        I2        = as.numeric(v)
    )
}
```

### Interpreting the Heterogeneity {#sec-heterogeneity-narrative}

The combined models show that mate-choice copying effect sizes are **highly heterogeneous**: total $I^2$ reaches `r fmt(i2get("Hedges' g", "Combined", "Total"), 1)`% for Hedges' $g$ and `r fmt(i2get("Log OR", "Combined", "Total"), 1)`% for the log odds ratio. In other words, the large majority of the observed variation reflects genuine differences among effects rather than sampling error alone, so the pooled means should be read as averages over a broad distribution of true effects rather than as a single common effect.

Decomposing that heterogeneity is where the phylogeny becomes informative. For the combined Hedges' $g$ model, the **phylogenetic** component accounts for `r fmt(i2get("Hedges' g", "Combined", "Phylogeny"), 1)`% of the variance, the **non-phylogenetic species** component for `r fmt(i2get("Hedges' g", "Combined", "Species (non-phylogenetic)"), 1)`%, between-**study** differences for `r fmt(i2get("Hedges' g", "Combined", "Study"), 1)`%, and the residual **effect-size** level for `r fmt(i2get("Hedges' g", "Combined", "Effect size (residual)"), 1)`%. The combined log-odds-ratio model partitions similarly: phylogeny `r fmt(i2get("Log OR", "Combined", "Phylogeny"), 1)`%, species `r fmt(i2get("Log OR", "Combined", "Species (non-phylogenetic)"), 1)`%, study `r fmt(i2get("Log OR", "Combined", "Study"), 1)`%, and residual `r fmt(i2get("Log OR", "Combined", "Effect size (residual)"), 1)`%.

The size of the phylogenetic component is the key new result. A substantial value would indicate that closely related species tend to show similar degrees of mate-choice copying — i.e., the trait carries a phylogenetic signal and effects are not independent across the tree — which would caution against treating species as exchangeable and would motivate phylogenetically informed interpretation. A small phylogenetic component, by contrast, indicates that relatedness explains little of the among-species variation: most of the species-level differences are idiosyncratic (captured by the non-phylogenetic species term) rather than inherited, and the earlier non-phylogenetic models were not materially biased by ignoring the tree. The contrast between the phylogenetic and non-phylogenetic species components is therefore the most direct readout of how much shared ancestry, as opposed to species-specific biology, structures the data.

Two further patterns are worth noting. First, the residual effect-size level typically absorbs a large share of the variance, reflecting heterogeneity among individual trials within studies (different paradigms, demonstrators, and conditions). Second, comparing the New-Only, Original-Only, and Combined columns shows whether the new extraction introduces or dilutes heterogeneity relative to the legacy datasets; where the new and original subsets allocate variance differently, the combined estimate reflects a weighted blend of the two. Because this high heterogeneity is the rule rather than the exception, it justifies the moderator and sub-group analyses pursued elsewhere in the workflow, and it is the reason the pooled estimates are reported with prediction intervals in addition to confidence intervals.

## Sensitivity Analysis: Impact of the VCV Structure

To evaluate the robustness of our meta-analytic results, we conduct a sensitivity analysis comparing the main multilevel models (which incorporate a Variance-Covariance [VCV] matrix with an assumed within-study correlation $\rho = 0.5$) against simplified multilevel models that ignore this correlation (i.e., treating effect size variances as independent diagonal elements, $V = v_i$).

Ignoring within-study correlation can lead to improper weighting of studies with multiple effect sizes and can bias the standard errors and confidence intervals of the pooled estimates.

```{r}
#| label: vcv_sensitivity_analysis
#| message: false
#| warning: false
# 1. Define comparison function
compare_vcv_models <- function(dat, name, yi = "yi", vi = "vi", cluster_var = "identifierStudyId", obs_var = "identifierEffectSizeID", rho = 0.5) {
    dat2 <- dat %>%
        filter(!is.na(.data[[yi]]), !is.na(.data[[vi]]), .data[[vi]] > 0)

    if (nrow(dat2) == 0) {
        return(NULL)
    }

    # A. With VCV (rho = 0.5)
    VCV <- metafor::vcalc(
        vi = dat2[[vi]],
        cluster = dat2[[cluster_var]],
        obs = dat2[[obs_var]],
        rho = rho,
        data = dat2
    )
    mod_w_vcv <- metafor::rma.mv(
        yi = dat2[[yi]],
        V = VCV,
        random = list(
            ~ 1 | identifierStudyId,
            ~ 1 | identifierEffectSizeID,
            ~ 1 | taxonomySpecies
        ),
        test = "t",
        method = "REML",
        sparse = TRUE,
        data = dat2
    )

    # B. Without VCV (diagonal V = vi)
    mod_no_vcv <- metafor::rma.mv(
        yi = dat2[[yi]],
        V = dat2[[vi]],
        random = list(
            ~ 1 | identifierStudyId,
            ~ 1 | identifierEffectSizeID,
            ~ 1 | taxonomySpecies
        ),
        test = "t",
        method = "REML",
        sparse = TRUE,
        data = dat2
    )

    # Combine results
    data.frame(
        Analysis = name,
        Model_Type = c("With VCV (rho = 0.5)", "Without VCV (Independent)"),
        k = c(mod_w_vcv$k, mod_no_vcv$k),
        Estimate = c(mod_w_vcv$b[1], mod_no_vcv$b[1]),
        SE = c(mod_w_vcv$se[1], mod_no_vcv$se[1]),
        CI_Lower = c(mod_w_vcv$ci.lb[1], mod_no_vcv$ci.lb[1]),
        CI_Upper = c(mod_w_vcv$ci.ub[1], mod_no_vcv$ci.ub[1]),
        p_val = c(mod_w_vcv$pval[1], mod_no_vcv$pval[1])
    )
}
```

### Key Methodological Findings

1. **Marginal Shifts in Pooled Estimates:** 
   Treating effect sizes as completely independent (Without VCV) causes very minor changes to the overall pooled estimates. For the combined dataset, the Hedges' $g$ estimate shifts from $`r fmt(comp_hedges_comb$Estimate[1])`$ to $`r fmt(comp_hedges_comb$Estimate[2])`$, while the Log Odds Ratio (lnOR) estimate shifts from $`r fmt(comp_or_comb$Estimate[1])`$ to $`r fmt(comp_or_comb$Estimate[2])`$. This indicates that the main directional conclusions of the meta-analyses are robust to VCV specification.

2. **Standard Error Shifts:** 
   In this multilevel model specification (which partitions variance using nested random intercepts), omitting the VCV matrix and assuming independence leads to a *slight inflation* of the standard error and wider confidence intervals for the combined analyses:
   * **Hedges' $g$ (Combined):** SE increases from $`r fmt(comp_hedges_comb$SE[1])`$ to $`r fmt(comp_hedges_comb$SE[2])`$ (CI width increases from $`r fmt(comp_hedges_comb$CI_Upper[1] - comp_hedges_comb$CI_Lower[1])`$ to $`r fmt(comp_hedges_comb$CI_Upper[2] - comp_hedges_comb$CI_Lower[2])`$).
   * **Log Odds Ratio (Combined):** SE increases from $`r fmt(comp_or_comb$SE[1])`$ to $`r fmt(comp_or_comb$SE[2])`$ (CI width increases from $`r fmt(comp_or_comb$CI_Upper[1] - comp_or_comb$CI_Lower[1])`$ to $`r fmt(comp_or_comb$CI_Upper[2] - comp_or_comb$CI_Lower[2])`$).
   
   This demonstrates that accounting for known within-study covariance via `vcalc` yields more precise, correctly-weighted estimates.

3. **Impact on Statistical Significance:**
   For continuous outcomes (Hedges' $g$ Combined), the pooled estimate is `r sigw(comp_hedges_comb$p_val[1])` with the VCV matrix ($`r pstr(comp_hedges_comb$p_val[1])`$) and `r sigw(comp_hedges_comb$p_val[2])` without it ($`r pstr(comp_hedges_comb$p_val[2])`$). Binary outcomes (Log OR Combined) are `r sigw(comp_or_comb$p_val[1])` under both specifications ($`r pstr(comp_or_comb$p_val[1])`$ with VCV; $`r pstr(comp_or_comb$p_val[2])`$ without).

Therefore, we conclude that while the overall direction and magnitude of the estimates are stable, **incorporating the VCV matrix** ensures appropriate weighting, controlling for within-study correlation, and obtaining accurate standard errors and statistical tests.
