Why this chapter exists

The Cross-Dataset Extraction Validation chapter compared the two original datasets that feed our combined synthesis — Davies et al. (2020) (continuous, Hedges’ d) and Jones & DuVal (2019) (binary, log odds ratio) — and returned to the source PDFs for the seven studies whose effect sizes disagreed by more than 0.5 Hedges-d units. That re-extraction reached two conclusions that matter for the headline result:

  1. On the continuous (Davies) side, the disagreements trace to a handful of identifiable extraction problems (a wrong outcome construct, two escalc computational artifacts, a sign error, and one defensible pool-vs-split). Several of these inflate the Davies effect size.
  2. On the binary (Jones) side, the point estimates are sound, but a few rows are over-counted (one spurious value and one duplicated contrast), and the variances of the single-group “vs-chance” studies are inflated by a phantom 50/50 control.

A natural question follows: does the combined overall effect depend on these contentious effect sizes? This chapter answers it directly. We re-fit the combined multilevel models — using the same model structure as the Overall Effect chapter — under three versions of the data and compare the pooled estimates.

  • Main — the data exactly as analysed in the overall-effect chapter (this should reproduce the headline combined estimate).
  • Leave-out — every effect size from the seven contentious studies is removed. This is the conservative bound: it asks what the result would be if we simply did not trust those studies at all.
  • Corrected — the contentious effect sizes are replaced with the design-faithful values re-extracted from the source PDFs. This is the best-estimate version: it keeps every flagged study but fixes the specific problems the validation identified. Note that the effect-size count changes slightly relative to Main, because correction is not a one-to-one value swap: on the binary side three over-counted rows are removed (the spurious DUG1992 value and the duplicated GIE2018 contrast), and on the continuous side Applebaum’s affiliation-time effects are not reinstated while Fowler-Finn’s pooled effect is split into two arm-level effects (see the edit map below).
NoteScope

The seven contentious studies (Applebaum & Cruz 2000, Briggs et al. 1996, Dugatkin 1992, Dugatkin & Godin 1993, Fowler-Finn et al. 2015, Gierszewski et al. 2018, Howard et al. 1998) are all pre-2019 and were not independently re-extracted in our new (Santos) dataset. They therefore enter the combined data only through the Davies (continuous) and Jones (binary) datasets, so every edit below acts on legacy rows only — the new extraction is untouched.

Identifying the contentious effect sizes

We read the re-extraction file produced by the validation chapter. Each of its rows is one effect size that we returned to the source PDF for, and it carries three things side by side: the re-extracted value (independent, design-faithful), the Jones value, and the Davies value. From this file alone we can locate every row that needs editing in the two combined datasets — by identifierEffectSizeID, never by guesswork.

reext_path <- here::here("outputs", "3_original_dataset_crosscheck", "reextraction_flagged_7studies.csv")
reext <- readr::read_csv(reext_path, show_col_types = FALSE)

# Conversion constant: lnOR -> Hedges' d (logistic-to-normal; Borenstein et al. 2009).
# Matches the validation chapter; the small-sample (J) correction is ~1 at these
# sample sizes and is omitted so the numbers agree with that chapter.
LNOR_TO_D  <- sqrt(3) / pi          # ~ 0.5513
VAR_LNOR_TO_VAR_D <- 3 / pi^2       # = LNOR_TO_D^2 ~ 0.3040

reext_map <- reext %>%
    transmute(
        study          = identifierStudyId,
        # Jones effect-size code in the dataset has no slash (e.g. "APP2000THE/1" -> "APP2000THE1")
        jones_es_id    = str_remove(compare_jones_code, "/"),
        # Davies effect-size number, parsed from e.g. "Davies study 3 (3.1)" -> "3.1"
        davies_es_id   = str_match(compare_davies_es, "\\(([0-9]+\\.[0-9]+)")[, 2],
        davies_study_no = str_match(compare_davies_es, "study ([0-9]+)")[, 2],
        species_phylo  = str_squish(as.character(taxonomySpecies)),
        # Re-extracted values
        reext_lnOR     = log(effectSizeCalculatedEstimate),
        reext_var_lnOR = effectSizeCalculatedVariance,
        reext_d        = log(effectSizeCalculatedEstimate) * LNOR_TO_D,
        reext_var_d    = effectSizeCalculatedVariance * VAR_LNOR_TO_VAR_D,
        # Prior values (for the audit table)
        jones_lnOR     = compare_jones_lnOR,
        jones_var      = compare_jones_VlnOR,

Re-fitting the combined models

We use the same multilevel model as the overall-effect chapter: rma.mv with a variance–covariance matrix (vcalc, within-study \(\rho = 0.5\)), random intercepts for study, effect size, and species, and a phylogenetic random effect supplied with the species correlation matrix (it falls back to the non-phylogenetic structure if a species is not in the matrix).

Show R code 
fit_overall_mv <- function(dat, cluster_var = "identifierStudyId",
                           obs_var = "identifierEffectSizeID", rho = 0.5,
                           phylo_cor = NULL, phylo_var = "species_phylo") {
    dat2 <- dat %>% filter(!is.na(yi), !is.na(vi), vi > 0)
    if (nrow(dat2) == 0) return(NULL)

    VCV <- metafor::vcalc(vi = dat2$vi, cluster = dat2[[cluster_var]], obs = dat2[[obs_var]],
                          rho = rho, data = dat2)

    random_list <- list(~ 1 | identifierStudyId, ~ 1 | identifierEffectSizeID, ~ 1 | taxonomySpecies)
    R_arg <- NULL
    if (!is.null(phylo_cor) && phylo_var %in% colnames(dat2)) {
        sp <- unique(as.character(dat2[[phylo_var]]))
        if (length(setdiff(sp, rownames(phylo_cor))) == 0) {
            dat2[[phylo_var]] <- factor(as.character(dat2[[phylo_var]]))
            Csub <- phylo_cor[sp, sp, drop = FALSE]
            random_list <- c(random_list, list(stats::as.formula(paste0("~ 1 | ", phylo_var))))
            R_arg <- stats::setNames(list(Csub), phylo_var)
        }
    }
    mod <- metafor::rma.mv(yi = dat2$yi, V = VCV, random = random_list, R = R_arg,
                           test = "t", method = "REML", sparse = TRUE, data = dat2)
    list(data = dat2, model = mod)
}

Comparison plot

We display each version as its own orchard trunk — the same plot format used elsewhere in this book. Each row is one model version (Main, Leave-out, Corrected); the bubbles are that version’s raw effect sizes (scaled by precision), the central marker is the pooled mean with its 95% confidence interval, and the thin whisker is the 95% prediction interval. The Mean [95% CI] and k (n) annotations sit beside each trunk.

Because the seven contentious studies are only a small fraction of the combined data, the three bubble clouds look broadly similar — that near-overlap is itself the robustness result. To make the edits legible, bubbles from the seven contentious studies are coloured orange and all other effect sizes grey. You can then see the contentious points present in Main, absent in Leave-out, and repositioned (and two fewer, after the over-count drops) in Corrected.

# Flag, for a fitted model's data, which effect sizes come from the seven contentious
# studies. For Hedges' g these are the flagged Davies rows plus the re-extracted
# replacements; for the log OR they are the flagged Jones rows. The flag is computed in
# the model's own row order, which orchaRd::mod_results preserves.
contentious_flag <- function(fit, class) {
    d   <- fit$data
    id  <- as.character(d$identifierEffectSizeID)
    src <- as.character(d$Dataset_Source)
    if (class == "hedges") {
        is_reext  <- grepl("^reext_", id)
        davies_no <- suppressWarnings(as.integer(str_extract(id, "^[0-9]+")))
        is_flag   <- src == "Original (Davies 2020)" & davies_no %in% davies_studies_flagged
        flag <- is_reext | (is_flag & !is.na(is_flag))
    } else {
        prefix <- str_extract(id, "^[A-Za-z]+[0-9]{4}")
        flag <- src == "Original (Jones & DuVal 2019)" & prefix %in% jones_papers_flagged
    }
    ifelse(flag, "Contentious study", "Other studies")
}

# Build one orchard "trunk" (an orchaRd mod_results object) from a fitted model and
# relabel it with the version name, mirroring the multi-trunk approach in overall_effect.qmd.
mk_trunk <- function(fit, name) {
    r <- orchaRd::mod_results(fit$model, group = "identifierStudyId")
    class(r) <- "list"
    r$mod_table$name <- name
    r$data$moderator <- name
    class(r) <- c("orchard", "data.frame")
    r
}

# Place "Mean [95% CI]" above the existing "k = .. (..)" label on each trunk
# (identical helper to the overall-effect chapter).
add_ci_labels <- function(plot_obj, orchard_results) {
    est  <- sprintf("%.2f", orchard_results$mod_table$estimate)
    ci_l <- sprintf("%.2f", orchard_results$mod_table$lowerCL)
    ci_u <- sprintf("%.2f", orchard_results$mod_table$upperCL)
    orig_labels <- plot_obj$layers[[6]]$aes_params$label
    new_labels <- character(0)
    for (i in seq_along(orig_labels)) {
        nums <- regmatches(orig_labels[i], gregexpr("[0-9]+", orig_labels[i]))[[1]]
        k_text <- if (length(nums) >= 2) paste0("k = ", nums[1], " (", nums[2], ")") else orig_labels[i]
        new_labels[i] <- paste0(est[i], " [", ci_l[i], ", ", ci_u[i], "]\n", k_text)
    }
    plot_obj$layers[[6]]$aes_params$label <- new_labels
    plot_obj$layers[[6]]$geom_params$parse <- FALSE
    plot_obj$layers[[6]]$aes_params$hjust <- "right"
    plot_obj$layers[[6]]$aes_params$vjust <- 0.5
    plot_obj$layers[[6]]$aes_params$size <- 3.0
    plot_obj
}

# Assemble a three-trunk orchard plot (Main / Leave-out / Corrected) for one class, with
# the raw bubbles recoloured by whether they come from a contentious study.
sensitivity_orchard <- function(fit_main, fit_leave, fit_corr, class, xlab, title) {
    t_main <- mk_trunk(fit_main,  "Main")
    t_leav <- mk_trunk(fit_leave, "Leave-out")
    t_corr <- mk_trunk(fit_corr,  "Corrected")
    sub <- orchaRd::submerge(t_main, t_leav, t_corr)

    # Attach the contentious flag in submerge order (Main, then Leave-out, then Corrected).
    sub$data$es_flag <- c(
        contentious_flag(fit_main,  class),
        contentious_flag(fit_leave, class),
        contentious_flag(fit_corr,  class)
    )

    p <- orchaRd::orchard_plot(sub, xlab = xlab, flip = TRUE)
    # Recolour the raw-point layer (layer 1) by the contentious flag.
    p$layers[[1]]$mapping$colour <- ggplot2::aes(colour = es_flag)$colour
    p$layers[[1]]$mapping$fill   <- ggplot2::aes(fill = es_flag)$fill

    p <- p +
        ggplot2::scale_colour_manual(values = flag_cols, name = NULL) +
        ggplot2::scale_fill_manual(values = flag_cols, name = NULL) +

Leave-one-out diagnostics

The analysis above asks a targeted question — does the headline result lean on the seven studies the cross-dataset validation flagged? A complementary, untargeted check is the classic leave-one-out diagnostic: re-fit the combined model many times, each time dropping one unit of data, and see whether any single unit moves the pooled estimate enough to change the story. If the leave-one-out means all stay tight around the full-data estimate (and inside its confidence band), no single study or species is driving the synthesis. We run it two ways, using the leave_one_out() and orchard_leave1out() tools in the orchaRd package (Nakagawa et al. 2023):

  • By study — drop all effect sizes from one study at a time. This is the standard influence check: it flags any individual paper the pooled estimate hinges on.
  • By species — drop all effect sizes from one species at a time. Because mate-choice copying is unevenly sampled across the tree (a few species, such as the guppy, contribute many effects), this asks whether the synthesis is really a statement about animals in general or an artefact of one well-studied taxon.

Each leave-one-out fit uses the same multilevel structure as the Main models above — the variance–covariance matrix is recomputed with vcalc (within-study \(\rho = 0.5\)) on each reduced dataset, and the phylogenetic random effect is retained, with the species correlation matrix subset to the species that remain. In each plot below, each row is one re-fit (labelled by the omitted study or species): grey filled dots are the effect sizes retained in that fit (scaled by precision), open orange rings are the effect sizes left out, and the white marker is the re-estimated pooled mean with its 95% confidence interval. The two horizontal red lines mark the full-data 95% confidence interval, so a mean sitting between them means leaving that unit out barely changes the synthesis. Rows are ordered by the leave-one-out mean (smallest at the bottom, largest at the top), which turns each panel into an influence ranking — the most consequential omissions sit at the extremes.

TipPlot styling

By default orchard_leave1out() orders rows by their order of appearance in the data and colours every bubble by which unit was left out — a purely decorative mapping (each row is already labelled) that, on the dark viridis scale used for >20 groups, hides both the pooled mean and the open-vs-filled distinction between retained and omitted effects. The style_loo() helper below strips that mapping, re-orders rows by estimate, and uses the grey-filled / open-orange encoding instead. Each panel is also written to outputs/1_effect_size_calculation_pipeline/ as both PNG and PDF (loo_study_hedges, loo_study_or, loo_species_hedges, loo_species_or) for use as manuscript figures.

# A leave-one-out-friendly refit of the Main model. It is identical in structure to
# fit_overall_mv() above, but is written so that orchaRd::leave_one_out() can rebuild and
# re-evaluate the model on each reduced dataset: the fitted objects (random-effects list,
# V matrix, data, and phylogenetic R matrix) are embedded as literals into the stored
# call, so the refit does not depend on this function's local variables.
fit_overall_loo <- function(dat, phylo_cor) {
    d <- dat %>% dplyr::filter(!is.na(yi), !is.na(vi), vi > 0)
    d$identifierStudyId      <- as.character(d$identifierStudyId)
    d$identifierEffectSizeID <- as.character(d$identifierEffectSizeID)
    d$taxonomySpecies        <- as.character(d$taxonomySpecies)
    d$species_phylo          <- factor(as.character(d$species_phylo))
    sp <- levels(d$species_phylo)
    use_phylo <- !is.null(phylo_cor) && length(setdiff(sp, rownames(phylo_cor))) == 0

    VCV  <- metafor::vcalc(vi = vi, cluster = identifierStudyId, obs = identifierEffectSizeID,
                           rho = 0.5, data = d)
    rand <- list(~ 1 | identifierStudyId, ~ 1 | identifierEffectSizeID, ~ 1 | taxonomySpecies)
    Rarg <- NULL
    if (use_phylo) {
        rand <- c(rand, list(~ 1 | species_phylo))
        Rarg <- list(species_phylo = phylo_cor[sp, sp, drop = FALSE])
    }
    mod <- metafor::rma.mv(yi = yi, V = VCV, random = rand, R = Rarg,
                           test = "t", method = "REML", sparse = TRUE, data = d)
    # Embed objects as literals so leave_one_out() can re-evaluate the call on each subset.
    mod$call$random <- rand
    mod$call$V      <- VCV
    mod$call$data   <- d
    if (use_phylo) mod$call$R <- Rarg
    mod
}

# This wrapper instead: re-orders the rows by the leave-one-out estimate (so the panel reads
# as an influence ranking); drops the decorative viridis mapping and shows the *retained*

# (Layer indices are fixed by orchaRd 2.2.0: 1 = retained bubbles, 3 = CI, 5 = mean, 8 = left-out.)
style_loo <- function(loo, xlab, base_size = 11) {

    ord <- as.character(loo$mod_table$name)[order(loo$mod_table$estimate)]
    lev <- c("Retained in fit", "Left out")

    # key; drop the decorative viridis colour/fill mapping.

    p$layers[[1]]$aes_params$shape <- 16
    p$layers[[1]]$aes_params$alpha <- 0.40

    p$layers[[8]]$aes_params$shape  <- 1
    p$layers[[8]]$aes_params$alpha  <- 0.85

    p$layers[[5]]$aes_params$size   <- 2.6
    p$layers[[5]]$aes_params$stroke <- 0.8
    # Re-stack so the open left-out rings sit BEHIND the CI/PI whisker and the mean marker.
    # Original draw order: 1 retained, 2 zero-line, 3 CI, 4 PI, 5 mean, 6 k-labels,
    # 7 reference-CI lines, 8 left-out -> move 8 up front (behind the summary geoms).
    p$layers <- p$layers[c(2, 1, 8, 3, 4, 7, 5, 6)]
    # Move the "k = .. (n)" labels into a clear right margin: left-justify them just past the
    # widest bubble and expand the axis to make room, so they no longer overlap the points.
    es     <- loo$data$yi
    ymax   <- max(c(es, loo$mod_table$upperPR), na.rm = TRUE)

        ggplot2::theme(legend.position = "right",

}

## Why this chapter exists

The [Cross-Dataset Extraction Validation](../3_original_dataset_crosscheck/extraction_validation.qmd)
chapter compared the two original datasets that feed our combined synthesis —
Davies et al. [-@daviesMetaanalysisFactorsInfluencing2020] (continuous, Hedges' *d*)
and Jones & DuVal [-@jonesMechanismsSocialInfluence2019] (binary, log odds ratio) —
and returned to the source PDFs for the **seven studies whose effect sizes disagreed
by more than 0.5 Hedges-*d* units**. That re-extraction reached two conclusions that
matter for the headline result:

1. On the **continuous (Davies) side**, the disagreements trace to a handful of
   identifiable extraction problems (a wrong outcome construct, two `escalc`
   computational artifacts, a sign error, and one defensible pool-vs-split). Several
   of these inflate the Davies effect size.
2. On the **binary (Jones) side**, the *point estimates* are sound, but a few rows are
   over-counted (one spurious value and one duplicated contrast), and the variances of
   the single-group "vs-chance" studies are inflated by a phantom 50/50 control.

A natural question follows: **does the combined overall effect depend on these
contentious effect sizes?** This chapter answers it directly. We re-fit the combined
multilevel models — using the *same* model structure as the
[Overall Effect](overall_effect.qmd) chapter — under three versions of the data and
compare the pooled estimates.

* **Main** — the data exactly as analysed in the overall-effect chapter (this should
  reproduce the headline combined estimate).
* **Leave-out** — every effect size from the seven contentious studies is removed. This
  is the conservative bound: it asks what the result would be if we simply did not trust
  those studies at all.
* **Corrected** — the contentious effect sizes are replaced with the design-faithful
  values re-extracted from the source PDFs. This is the best-estimate version: it keeps
  every flagged *study* but fixes the specific problems the validation identified. Note
  that the *effect-size* count changes slightly relative to **Main**, because correction
  is not a one-to-one value swap: on the binary side three over-counted rows are removed
  (the spurious DUG1992 value and the duplicated GIE2018 contrast), and on the continuous
  side Applebaum's affiliation-time effects are not reinstated while Fowler-Finn's pooled
  effect is split into two arm-level effects (see the edit map below).

## Identifying the contentious effect sizes

We read the re-extraction file produced by the validation chapter. Each of its rows is
one effect size that we returned to the source PDF for, and it carries three things side
by side: the **re-extracted** value (independent, design-faithful), the **Jones** value,
and the **Davies** value. From this file alone we can locate every row that needs editing
in the two combined datasets — by `identifierEffectSizeID`, never by guesswork.

## Re-fitting the combined models

We use the **same** multilevel model as the overall-effect chapter: `rma.mv` with a
variance–covariance matrix (`vcalc`, within-study $\rho = 0.5$), random intercepts for
study, effect size, and species, and a phylogenetic random effect supplied with the
species correlation matrix (it falls back to the non-phylogenetic structure if a species
is not in the matrix).

```{r}
#| label: fit_function
#| message: false
#| warning: false
fit_overall_mv <- function(dat, cluster_var = "identifierStudyId",
                           obs_var = "identifierEffectSizeID", rho = 0.5,
                           phylo_cor = NULL, phylo_var = "species_phylo") {
    dat2 <- dat %>% filter(!is.na(yi), !is.na(vi), vi > 0)
    if (nrow(dat2) == 0) return(NULL)

    VCV <- metafor::vcalc(vi = dat2$vi, cluster = dat2[[cluster_var]], obs = dat2[[obs_var]],
                          rho = rho, data = dat2)

    random_list <- list(~ 1 | identifierStudyId, ~ 1 | identifierEffectSizeID, ~ 1 | taxonomySpecies)
    R_arg <- NULL
    if (!is.null(phylo_cor) && phylo_var %in% colnames(dat2)) {
        sp <- unique(as.character(dat2[[phylo_var]]))
        if (length(setdiff(sp, rownames(phylo_cor))) == 0) {
            dat2[[phylo_var]] <- factor(as.character(dat2[[phylo_var]]))
            Csub <- phylo_cor[sp, sp, drop = FALSE]
            random_list <- c(random_list, list(stats::as.formula(paste0("~ 1 | ", phylo_var))))
            R_arg <- stats::setNames(list(Csub), phylo_var)
        }
    }
    mod <- metafor::rma.mv(yi = dat2$yi, V = VCV, random = random_list, R = R_arg,
                           test = "t", method = "REML", sparse = TRUE, data = dat2)
    list(data = dat2, model = mod)
}
```

### Comparison plot

We display each version as its own orchard trunk — the same plot format used elsewhere in
this book. Each row is one model version (Main, Leave-out, Corrected); the bubbles are that
version's raw effect sizes (scaled by precision), the central marker is the pooled mean with
its 95% confidence interval, and the thin whisker is the 95% prediction interval. The
`Mean [95% CI]` and `k (n)` annotations sit beside each trunk.

Because the seven contentious studies are only a small fraction of the combined data, the
three bubble clouds look broadly similar — that near-overlap is itself the robustness
result. To make the edits legible, **bubbles from the seven contentious studies are coloured
orange and all other effect sizes grey.** You can then see the contentious points present in
**Main**, *absent* in **Leave-out**, and *repositioned* (and two fewer, after the over-count
drops) in **Corrected**.

```{r}
#| label: orchard_helpers
#| message: false
#| warning: false
# Flag, for a fitted model's data, which effect sizes come from the seven contentious
# studies. For Hedges' g these are the flagged Davies rows plus the re-extracted
# replacements; for the log OR they are the flagged Jones rows. The flag is computed in
# the model's own row order, which orchaRd::mod_results preserves.
contentious_flag <- function(fit, class) {
    d   <- fit$data
    id  <- as.character(d$identifierEffectSizeID)
    src <- as.character(d$Dataset_Source)
    if (class == "hedges") {
        is_reext  <- grepl("^reext_", id)
        davies_no <- suppressWarnings(as.integer(str_extract(id, "^[0-9]+")))
        is_flag   <- src == "Original (Davies 2020)" & davies_no %in% davies_studies_flagged
        flag <- is_reext | (is_flag & !is.na(is_flag))
    } else {
        prefix <- str_extract(id, "^[A-Za-z]+[0-9]{4}")
        flag <- src == "Original (Jones & DuVal 2019)" & prefix %in% jones_papers_flagged
    }
    ifelse(flag, "Contentious study", "Other studies")
}

flag_cols <- c("Contentious study" = "#D55E00", "Other studies" = "grey70")

# Build one orchard "trunk" (an orchaRd mod_results object) from a fitted model and
# relabel it with the version name, mirroring the multi-trunk approach in overall_effect.qmd.
mk_trunk <- function(fit, name) {
    r <- orchaRd::mod_results(fit$model, group = "identifierStudyId")
    class(r) <- "list"
    r$mod_table$name <- name
    r$data$moderator <- name
    class(r) <- c("orchard", "data.frame")
    r
}

# Place "Mean [95% CI]" above the existing "k = .. (..)" label on each trunk
# (identical helper to the overall-effect chapter).
add_ci_labels <- function(plot_obj, orchard_results) {
    est  <- sprintf("%.2f", orchard_results$mod_table$estimate)
    ci_l <- sprintf("%.2f", orchard_results$mod_table$lowerCL)
    ci_u <- sprintf("%.2f", orchard_results$mod_table$upperCL)
    orig_labels <- plot_obj$layers[[6]]$aes_params$label
    new_labels <- character(0)
    for (i in seq_along(orig_labels)) {
        nums <- regmatches(orig_labels[i], gregexpr("[0-9]+", orig_labels[i]))[[1]]
        k_text <- if (length(nums) >= 2) paste0("k = ", nums[1], " (", nums[2], ")") else orig_labels[i]
        new_labels[i] <- paste0(est[i], " [", ci_l[i], ", ", ci_u[i], "]\n", k_text)
    }
    plot_obj$layers[[6]]$aes_params$label <- new_labels
    plot_obj$layers[[6]]$geom_params$parse <- FALSE
    plot_obj$layers[[6]]$aes_params$hjust <- "right"
    plot_obj$layers[[6]]$aes_params$vjust <- 0.5
    plot_obj$layers[[6]]$aes_params$size <- 3.0
    plot_obj
}

# Assemble a three-trunk orchard plot (Main / Leave-out / Corrected) for one class, with
# the raw bubbles recoloured by whether they come from a contentious study.
sensitivity_orchard <- function(fit_main, fit_leave, fit_corr, class, xlab, title) {
    t_main <- mk_trunk(fit_main,  "Main")
    t_leav <- mk_trunk(fit_leave, "Leave-out")
    t_corr <- mk_trunk(fit_corr,  "Corrected")
    sub <- orchaRd::submerge(t_main, t_leav, t_corr)

    # Attach the contentious flag in submerge order (Main, then Leave-out, then Corrected).
    sub$data$es_flag <- c(
        contentious_flag(fit_main,  class),
        contentious_flag(fit_leave, class),
        contentious_flag(fit_corr,  class)
    )

    p <- orchaRd::orchard_plot(sub, xlab = xlab, flip = TRUE)
    # Recolour the raw-point layer (layer 1) by the contentious flag.
    p$layers[[1]]$mapping$colour <- ggplot2::aes(colour = es_flag)$colour
    p$layers[[1]]$mapping$fill   <- ggplot2::aes(fill = es_flag)$fill

    p <- p +
        ggplot2::scale_colour_manual(values = flag_cols, name = NULL) +
        ggplot2::scale_fill_manual(values = flag_cols, name = NULL) +
        ggplot2::ggtitle(title) +
        ggplot2::theme(
            legend.position = "bottom",
            plot.title = ggplot2::element_text(face = "bold", hjust = 0.5)
        )
    add_ci_labels(p, sub)
}
```

```{r}
sig <- function(p) if (is.na(p)) "indeterminate" else if (p < 0.05) "statistically significant" else "non-significant"
```

**Continuous (Hedges' *g*).** The combined mean is $`r fmt(hm, 2)`$ in the **Main**
analysis (`r sig(hm_p)`), $`r fmt(hl, 2)`$ when the contentious studies are **left out**
(`r sig(hl_p)`), and $`r fmt(hc, 2)`$ when they are **corrected** (`r sig(hc_p)`). Because
the validation showed the Davies errors mostly *inflate* the effect, the leave-out and
corrected means are expected to move toward — not away from — zero relative to Main;
the size of that shift is the practical measure of how much the headline continuous
estimate leaned on the flagged extractions.

**Binary (log odds ratio).** The combined mean is $`r fmt(om, 2)`$ in **Main**
(`r sig(om_p)`), $`r fmt(ol, 2)`$ **leaving out** the contentious papers (`r sig(ol_p)`),
and $`r fmt(oc, 2)`$ when **corrected** (`r sig(oc_p)`). On the binary side the
corrections act mainly on *variances* (removing the phantom-control inflation) and remove
two over-counted rows, so the point estimate is expected to move less than the continuous
one; the main effect there is on precision and weighting rather than on the location of
the mean.

**Reading the three together.** The **Leave-out** column is the conservative bound (trust
nothing flagged); the **Corrected** column is the best estimate (keep the studies, fix the
errors). If both fall close to **Main** and keep the same sign and significance, the
overall conclusion is robust to the extraction problems the validation uncovered. If
**Main** sits apart from the other two, the headline estimate depended materially on the
contentious effect sizes, and the **Corrected** value should be preferred as the
bias-reduced summary.

## Leave-one-out diagnostics

The analysis above asks a *targeted* question — does the headline result lean on the
seven studies the cross-dataset validation flagged? A complementary, *untargeted* check
is the classic **leave-one-out** diagnostic: re-fit the combined model many times, each
time dropping **one** unit of data, and see whether any single unit moves the pooled
estimate enough to change the story. If the leave-one-out means all stay tight around the
full-data estimate (and inside its confidence band), no single study or species is driving
the synthesis. We run it two ways, using the `leave_one_out()` and `orchard_leave1out()`
tools in the `orchaRd` package [@nakagawaOrchaRd2Visualising2023]:

* **By study** — drop all effect sizes from one study at a time. This is the standard
  influence check: it flags any individual paper the pooled estimate hinges on.
* **By species** — drop all effect sizes from one species at a time. Because mate-choice
  copying is unevenly sampled across the tree (a few species, such as the guppy, contribute
  many effects), this asks whether the synthesis is really a statement about animals in
  general or an artefact of one well-studied taxon.

Each leave-one-out fit uses the **same** multilevel structure as the Main models above —
the variance–covariance matrix is recomputed with `vcalc` (within-study $\rho = 0.5$) on
each reduced dataset, and the phylogenetic random effect is retained, with the species
correlation matrix subset to the species that remain. In each plot below, each row is one
re-fit (labelled by the omitted study or species): **grey filled dots are the effect sizes
retained** in that fit (scaled by precision), **open orange rings are the effect sizes left
out**, and the **white marker is the re-estimated pooled mean with its 95% confidence
interval**. The two horizontal **red lines mark the full-data 95% confidence interval**, so
a mean sitting between them means leaving that unit out barely changes the synthesis.
**Rows are ordered by the leave-one-out mean** (smallest at the bottom, largest at the top),
which turns each panel into an influence ranking — the most consequential omissions sit at
the extremes.

::: {.callout-tip}
## Plot styling
By default `orchard_leave1out()` orders rows by their order of appearance in the data and
colours every bubble by *which* unit was left out — a purely decorative mapping (each row is
already labelled) that, on the dark viridis scale used for >20 groups, hides both the pooled
mean and the open-vs-filled distinction between retained and omitted effects. The `style_loo()`
helper below strips that mapping, re-orders rows by estimate, and uses the grey-filled /
open-orange encoding instead. Each panel is also written to `outputs/1_effect_size_calculation_pipeline/`
as both PNG and **PDF** (`loo_study_hedges`, `loo_study_or`, `loo_species_hedges`,
`loo_species_or`) for use as manuscript figures.
:::

```{r}
#| label: loo_fit_function
#| message: false
#| warning: false
# A leave-one-out-friendly refit of the Main model. It is identical in structure to
# fit_overall_mv() above, but is written so that orchaRd::leave_one_out() can rebuild and
# re-evaluate the model on each reduced dataset: the fitted objects (random-effects list,
# V matrix, data, and phylogenetic R matrix) are embedded as literals into the stored
# call, so the refit does not depend on this function's local variables.
fit_overall_loo <- function(dat, phylo_cor) {
    d <- dat %>% dplyr::filter(!is.na(yi), !is.na(vi), vi > 0)
    d$identifierStudyId      <- as.character(d$identifierStudyId)
    d$identifierEffectSizeID <- as.character(d$identifierEffectSizeID)
    d$taxonomySpecies        <- as.character(d$taxonomySpecies)
    d$species_phylo          <- factor(as.character(d$species_phylo))
    sp <- levels(d$species_phylo)
    use_phylo <- !is.null(phylo_cor) && length(setdiff(sp, rownames(phylo_cor))) == 0

    VCV  <- metafor::vcalc(vi = vi, cluster = identifierStudyId, obs = identifierEffectSizeID,
                           rho = 0.5, data = d)
    rand <- list(~ 1 | identifierStudyId, ~ 1 | identifierEffectSizeID, ~ 1 | taxonomySpecies)
    Rarg <- NULL
    if (use_phylo) {
        rand <- c(rand, list(~ 1 | species_phylo))
        Rarg <- list(species_phylo = phylo_cor[sp, sp, drop = FALSE])
    }
    mod <- metafor::rma.mv(yi = yi, V = VCV, random = rand, R = Rarg,
                           test = "t", method = "REML", sparse = TRUE, data = d)
    # Embed objects as literals so leave_one_out() can re-evaluate the call on each subset.
    mod$call$random <- rand
    mod$call$V      <- VCV
    mod$call$data   <- d
    if (use_phylo) mod$call$R <- Rarg
    mod
}

# Plot helper. orchaRd::orchard_leave1out() defaults to (a) data order for the rows,
# (b) a viridis_d palette (for >20 groups) that colours every bubble by *which* unit was
# left out — a decorative mapping, since each row is already labelled — and renders both
# the bubbles and the per-row mean in dark hues, and (c) an inside legend that collides with
# the bottom row. It also draws the retained and omitted ("ghost") effect sizes as the same
# filled circle, differing only in opacity, so they are hard to tell apart.
#
# This wrapper instead: re-orders the rows by the leave-one-out estimate (so the panel reads
# as an influence ranking); drops the decorative viridis mapping and shows the *retained*
# effect sizes as solid grey dots and the *left-out* effect sizes as open orange rings, with
# a legend that names them; draws each pooled mean as a white-filled black marker that stands
# out against the cloud; and moves the legends out of the panel.
# (Layer indices are fixed by orchaRd 2.2.0: 1 = retained bubbles, 3 = CI, 5 = mean, 8 = left-out.)
style_loo <- function(loo, xlab, base_size = 11) {
    p   <- orchaRd::orchard_leave1out(loo, xlab = xlab, angle = 0)
    ord <- as.character(loo$mod_table$name)[order(loo$mod_table$estimate)]
    lev <- c("Retained in fit", "Left out")
    # Retained effect sizes: solid grey, filled. Map colour to a constant so it gets a legend
    # key; drop the decorative viridis colour/fill mapping.
    p$layers[[1]]$mapping$fill   <- NULL
    p$layers[[1]]$mapping$colour <- ggplot2::aes(colour = "Retained in fit")$colour
    p$layers[[1]]$aes_params$shape <- 16
    p$layers[[1]]$aes_params$alpha <- 0.40
    # Left-out effect sizes: open rings, more opaque, distinct colour.
    p$layers[[8]]$mapping$colour <- ggplot2::aes(colour = "Left out")$colour
    p$layers[[8]]$aes_params$shape  <- 1
    p$layers[[8]]$aes_params$alpha  <- 0.85
    p$layers[[8]]$aes_params$stroke <- 0.6
    # CI bar and pooled-mean marker.
    p$layers[[3]]$aes_params$linewidth <- 1.1
    p$layers[[3]]$aes_params$colour    <- "black"
    p$layers[[5]]$mapping$fill   <- NULL
    p$layers[[5]]$aes_params$fill   <- "white"
    p$layers[[5]]$aes_params$colour <- "black"
    p$layers[[5]]$aes_params$size   <- 2.6
    p$layers[[5]]$aes_params$stroke <- 0.8
    # Re-stack so the open left-out rings sit BEHIND the CI/PI whisker and the mean marker.
    # Original draw order: 1 retained, 2 zero-line, 3 CI, 4 PI, 5 mean, 6 k-labels,
    # 7 reference-CI lines, 8 left-out -> move 8 up front (behind the summary geoms).
    p$layers <- p$layers[c(2, 1, 8, 3, 4, 7, 5, 6)]
    # Move the "k = .. (n)" labels into a clear right margin: left-justify them just past the
    # widest bubble and expand the axis to make room, so they no longer overlap the points.
    es     <- loo$data$yi
    ymax   <- max(c(es, loo$mod_table$upperPR), na.rm = TRUE)
    ymin   <- min(c(es, loo$mod_table$lowerPR), na.rm = TRUE)
    span   <- ymax - ymin
    anchor <- ymax + 0.06 * span
    lab_i  <- which(vapply(p$layers, function(l) inherits(l$geom, "GeomText"), logical(1)))
    p$layers[[lab_i]]$data$y           <- anchor
    p$layers[[lab_i]]$aes_params$hjust <- 0
    p +
        ggplot2::scale_x_discrete(limits = ord) +
        ggplot2::scale_y_continuous(limits = c(ymin, anchor + 0.34 * span)) +
        ggplot2::scale_colour_manual(values = c("Retained in fit" = "grey60", "Left out" = "#D55E00"),
                                     breaks = lev, name = NULL) +
        ggplot2::guides(colour = ggplot2::guide_legend(
            override.aes = list(shape = c(16, 1), alpha = 1, size = 3, stroke = 0.8))) +
        ggplot2::theme_bw(base_size = base_size) +
        ggplot2::theme(legend.position = "right",
                       panel.grid.minor = ggplot2::element_blank())
}
```
