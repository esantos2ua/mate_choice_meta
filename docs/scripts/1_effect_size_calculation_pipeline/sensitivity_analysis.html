How robust is the combined overall effect to the studies flagged by the cross-dataset validation?

Author

Eduardo S. A. Santos & Collaborators

Published

June 23, 2026

Why this chapter exists

The Cross-Dataset Extraction Validation chapter compared the two original datasets that feed our combined synthesis — Davies et al. (2020) (continuous, Hedges’ d) and Jones & DuVal (2019) (binary, log odds ratio) — and returned to the source PDFs for the seven studies whose effect sizes disagreed by more than 0.5 Hedges-d units. That re-extraction reached two conclusions that matter for the headline result:

  1. On the continuous (Davies) side, the disagreements trace to a handful of identifiable extraction problems (a wrong outcome construct, two escalc computational artifacts, a sign error, and one defensible pool-vs-split). Several of these inflate the Davies effect size.
  2. On the binary (Jones) side, the point estimates are sound, but a few rows are over-counted (one spurious value and one duplicated contrast), and the variances of the single-group “vs-chance” studies are inflated by a phantom 50/50 control.

A natural question follows: does the combined overall effect depend on these contentious effect sizes? This chapter answers it directly. We re-fit the combined multilevel models — using the same model structure as the Overall Effect chapter — under three versions of the data and compare the pooled estimates.

  • Main — the data exactly as analysed in the overall-effect chapter (this should reproduce the headline combined estimate).
  • Leave-out — every effect size from the seven contentious studies is removed. This is the conservative bound: it asks what the result would be if we simply did not trust those studies at all.
  • Corrected — the contentious effect sizes are replaced with the design-faithful values re-extracted from the source PDFs. This is the best-estimate version: it keeps the studies but fixes the specific problems the validation identified.
NoteScope

The seven contentious studies (Applebaum & Cruz 2000, Briggs et al. 1996, Dugatkin 1992, Dugatkin & Godin 1993, Fowler-Finn et al. 2015, Gierszewski et al. 2018, Howard et al. 1998) are all pre-2019 and were not independently re-extracted in our new (Santos) dataset. They therefore enter the combined data only through the Davies (continuous) and Jones (binary) datasets, so every edit below acts on legacy rows only — the new extraction is untouched.

Identifying the contentious effect sizes

We read the re-extraction file produced by the validation chapter. Each of its rows is one effect size that we returned to the source PDF for, and it carries three things side by side: the re-extracted value (independent, design-faithful), the Jones value, and the Davies value. From this file alone we can locate every row that needs editing in the two combined datasets — by identifierEffectSizeID, never by guesswork.

Show R code 
reext_path <- here::here("outputs", "3_original_dataset_crosscheck", "reextraction_flagged_7studies.csv")
reext <- readr::read_csv(reext_path, show_col_types = FALSE)

# Conversion constant: lnOR -> Hedges' d (logistic-to-normal; Borenstein et al. 2009).
# Matches the validation chapter; the small-sample (J) correction is ~1 at these
# sample sizes and is omitted so the numbers agree with that chapter.
LNOR_TO_D  <- sqrt(3) / pi          # ~ 0.5513
VAR_LNOR_TO_VAR_D <- 3 / pi^2       # = LNOR_TO_D^2 ~ 0.3040

reext_map <- reext %>%
    transmute(
        study          = identifierStudyId,
        # Jones effect-size code in the dataset has no slash (e.g. "APP2000THE/1" -> "APP2000THE1")
        jones_es_id    = str_remove(compare_jones_code, "/"),
        # Davies effect-size number, parsed from e.g. "Davies study 3 (3.1)" -> "3.1"
        davies_es_id   = str_match(compare_davies_es, "\\(([0-9]+\\.[0-9]+)")[, 2],
        davies_study_no = str_match(compare_davies_es, "study ([0-9]+)")[, 2],
        species_phylo  = str_squish(as.character(taxonomySpecies)),
        # Re-extracted values
        reext_lnOR     = log(effectSizeCalculatedEstimate),
        reext_var_lnOR = effectSizeCalculatedVariance,
        reext_d        = log(effectSizeCalculatedEstimate) * LNOR_TO_D,
        reext_var_d    = effectSizeCalculatedVariance * VAR_LNOR_TO_VAR_D,
        # Prior values (for the audit table)
        jones_lnOR     = compare_jones_lnOR,
        jones_var      = compare_jones_VlnOR,
        davies_d       = compare_davies_hedges_d,
        davies_var     = compare_davies_variance
    )

# The set of contentious studies, expressed in each dataset's own identifiers.
davies_studies_flagged <- sort(unique(as.integer(reext_map$davies_study_no)))   # Davies study numbers
jones_papers_flagged   <- sort(unique(str_extract(reext_map$jones_es_id, "^[A-Za-z]+[0-9]{4}")))  # Jones paper prefixes

# Jones rows the validation identified as over-counts to be dropped entirely:
#  * DUG1992REV1  — spurious third value (reproduces from no experiment pair)
#  * GIE2018MAT1/2 — duplicate of the GIE2018USI contrast (over-count)
jones_drop_overcount <- c("DUG1992REV1", "GIE2018MAT1", "GIE2018MAT2")

# ===========================================================================

             jones_paper(identifierEffectSizeID) %in% jones_papers_flagged))

# (2) CORRECTED: (a) drop the over-count rows; (b) replace lnOR & variance of the
#     re-extracted codes with the design-faithful values; leave all other Jones rows
#     (e.g. the legitimate, un-re-extracted APP2000THE2 baseline contrast) untouched.

    select(-reext_lnOR, -reext_var_lnOR)

Re-fitting the combined models

We use the same multilevel model as the overall-effect chapter: rma.mv with a variance–covariance matrix (vcalc, within-study \(\rho = 0.5\)), random intercepts for study, effect size, and species, and a phylogenetic random effect supplied with the species correlation matrix (it falls back to the non-phylogenetic structure if a species is not in the matrix).

Show R code 
fit_overall_mv <- function(dat, cluster_var = "identifierStudyId",
                           obs_var = "identifierEffectSizeID", rho = 0.5,
                           phylo_cor = NULL, phylo_var = "species_phylo") {
    dat2 <- dat %>% filter(!is.na(yi), !is.na(vi), vi > 0)
    if (nrow(dat2) == 0) return(NULL)

    VCV <- metafor::vcalc(vi = dat2$vi, cluster = dat2[[cluster_var]], obs = dat2[[obs_var]],
                          rho = rho, data = dat2)

    random_list <- list(~ 1 | identifierStudyId, ~ 1 | identifierEffectSizeID, ~ 1 | taxonomySpecies)
    R_arg <- NULL
    if (!is.null(phylo_cor) && phylo_var %in% colnames(dat2)) {
        sp <- unique(as.character(dat2[[phylo_var]]))
        if (length(setdiff(sp, rownames(phylo_cor))) == 0) {
            dat2[[phylo_var]] <- factor(as.character(dat2[[phylo_var]]))
            Csub <- phylo_cor[sp, sp, drop = FALSE]
            random_list <- c(random_list, list(stats::as.formula(paste0("~ 1 | ", phylo_var))))
            R_arg <- stats::setNames(list(Csub), phylo_var)
        }
    }
    mod <- metafor::rma.mv(yi = dat2$yi, V = VCV, random = random_list, R = R_arg,
                           test = "t", method = "REML", sparse = TRUE, data = dat2)
    list(data = dat2, model = mod)
}

# Fit all six models
fits <- list(
    hedges_main      = fit_overall_mv(db_hedges,        phylo_cor = phylo_cor_hedges),

# Pull a tidy one-row summary from each fitted model
summarise_fit <- function(fit, class_lab, version_lab) {
    if (is.null(fit)) return(NULL)
    m <- fit$model
    i2 <- tryCatch(sum(orchaRd::i2_ml(m)["I2_Total"]), error = function(e) NA_real_)
    tibble(
        Class = class_lab, Version = version_lab,
        k = m$k, Studies = length(unique(fit$data$identifierStudyId)),
        Estimate = as.numeric(m$b[1]), SE = m$se[1],
        CI_low = m$ci.lb[1], CI_high = m$ci.ub[1],
        t = m$zval[1], p = m$pval[1], I2 = i2
    )
}

Comparison plot

We display each version as its own orchard trunk — the same plot format used elsewhere in this book. Each row is one model version (Main, Leave-out, Corrected); the bubbles are that version’s raw effect sizes (scaled by precision), the central marker is the pooled mean with its 95% confidence interval, and the thin whisker is the 95% prediction interval. The Mean [95% CI] and k (n) annotations sit beside each trunk.

# Build one orchard "trunk" (an orchaRd mod_results object) from a fitted model and
# relabel it with the version name, mirroring the multi-trunk approach in overall_effect.qmd.
mk_trunk <- function(fit, name) {
    r <- orchaRd::mod_results(fit$model, group = "identifierStudyId")
    class(r) <- "list"
    r$mod_table$name <- name
    r$data$moderator <- name
    class(r) <- c("orchard", "data.frame")
    r
}

# Place "Mean [95% CI]" above the existing "k = .. (..)" label on each trunk
# (identical helper to the overall-effect chapter).
add_ci_labels <- function(plot_obj, orchard_results) {
    est  <- sprintf("%.2f", orchard_results$mod_table$estimate)
    ci_l <- sprintf("%.2f", orchard_results$mod_table$lowerCL)
    ci_u <- sprintf("%.2f", orchard_results$mod_table$upperCL)
    orig_labels <- plot_obj$layers[[6]]$aes_params$label
    new_labels <- character(0)
    for (i in seq_along(orig_labels)) {
        nums <- regmatches(orig_labels[i], gregexpr("[0-9]+", orig_labels[i]))[[1]]
        k_text <- if (length(nums) >= 2) paste0("k = ", nums[1], " (", nums[2], ")") else orig_labels[i]
        new_labels[i] <- paste0(est[i], " [", ci_l[i], ", ", ci_u[i], "]\n", k_text)
    }
    plot_obj$layers[[6]]$aes_params$label <- new_labels
    plot_obj$layers[[6]]$geom_params$parse <- FALSE
    plot_obj$layers[[6]]$aes_params$hjust <- "right"
    plot_obj$layers[[6]]$aes_params$vjust <- 0.5
    plot_obj$layers[[6]]$aes_params$size <- 3.0
    plot_obj
}

# Assemble a three-trunk orchard plot (Main / Leave-out / Corrected) for one class.
sensitivity_orchard <- function(fit_main, fit_leave, fit_corr, xlab, title) {
    t_main <- mk_trunk(fit_main,  "Main")
    t_leav <- mk_trunk(fit_leave, "Leave-out")
    t_corr <- mk_trunk(fit_corr,  "Corrected")
    sub <- orchaRd::submerge(t_main, t_leav, t_corr)

    p <- orchaRd::orchard_plot(sub, xlab = xlab, flip = TRUE) +
        ggplot2::scale_colour_manual(values = ver_cols, name = "Dataset version") +
        ggplot2::scale_fill_manual(values = ver_cols, name = "Dataset version") +
        ggplot2::ggtitle(title) +
        ggplot2::theme(
            legend.position = "none",
            plot.title = ggplot2::element_text(face = "bold", hjust = 0.5)
        )
    add_ci_labels(p, sub)
}

Interpretation

Continuous (Hedges’ g). The combined mean is \(0.45\) in the Main analysis (statistically significant), \(0.42\) when the contentious studies are left out (statistically significant), and \(0.42\) when they are corrected (statistically significant). Because the validation showed the Davies errors mostly inflate the effect, the leave-out and corrected means are expected to move toward — not away from — zero relative to Main; the size of that shift is the practical measure of how much the headline continuous estimate leaned on the flagged extractions.

Binary (log odds ratio). The combined mean is \(0.59\) in Main (statistically significant), \(0.57\) leaving out the contentious papers (statistically significant), and \(0.58\) when corrected (statistically significant). On the binary side the corrections act mainly on variances (removing the phantom-control inflation) and remove two over-counted rows, so the point estimate is expected to move less than the continuous one; the main effect there is on precision and weighting rather than on the location of the mean.

Reading the three together. The Leave-out column is the conservative bound (trust nothing flagged); the Corrected column is the best estimate (keep the studies, fix the errors). If both fall close to Main and keep the same sign and significance, the overall conclusion is robust to the extraction problems the validation uncovered. If Main sits apart from the other two, the headline estimate depended materially on the contentious effect sizes, and the Corrected value should be preferred as the bias-reduced summary.

WarningWhat this analysis does and does not show

This is a sensitivity analysis of the combined overall mean to the seven studies the cross-dataset validation flagged. It does not re-audit the rest of the literature, and it inherits every other modelling choice from the overall-effect chapter (the multilevel structure, the within-study \(\rho = 0.5\), and the lnOR\(\leftrightarrow\)d scale conversion). The “Corrected” edits are exactly those documented in the validation chapter and listed in the edit map above; no other values were changed.

## Why this chapter exists

The [Cross-Dataset Extraction Validation](../3_original_dataset_crosscheck/extraction_validation.qmd)
chapter compared the two original datasets that feed our combined synthesis —
Davies et al. [-@daviesMetaanalysisFactorsInfluencing2020] (continuous, Hedges' *d*)
and Jones & DuVal [-@jonesMechanismsSocialInfluence2019] (binary, log odds ratio) —
and returned to the source PDFs for the **seven studies whose effect sizes disagreed
by more than 0.5 Hedges-*d* units**. That re-extraction reached two conclusions that
matter for the headline result:

1. On the **continuous (Davies) side**, the disagreements trace to a handful of
   identifiable extraction problems (a wrong outcome construct, two `escalc`
   computational artifacts, a sign error, and one defensible pool-vs-split). Several
   of these inflate the Davies effect size.
2. On the **binary (Jones) side**, the *point estimates* are sound, but a few rows are
   over-counted (one spurious value and one duplicated contrast), and the variances of
   the single-group "vs-chance" studies are inflated by a phantom 50/50 control.

A natural question follows: **does the combined overall effect depend on these
contentious effect sizes?** This chapter answers it directly. We re-fit the combined
multilevel models — using the *same* model structure as the
[Overall Effect](overall_effect.qmd) chapter — under three versions of the data and
compare the pooled estimates.

* **Main** — the data exactly as analysed in the overall-effect chapter (this should
  reproduce the headline combined estimate).
* **Leave-out** — every effect size from the seven contentious studies is removed. This
  is the conservative bound: it asks what the result would be if we simply did not trust
  those studies at all.
* **Corrected** — the contentious effect sizes are replaced with the design-faithful
  values re-extracted from the source PDFs. This is the best-estimate version: it keeps
  the studies but fixes the specific problems the validation identified.

::: {.callout-note}
## Scope
The seven contentious studies (Applebaum & Cruz 2000, Briggs et al. 1996, Dugatkin 1992,
Dugatkin & Godin 1993, Fowler-Finn et al. 2015, Gierszewski et al. 2018, Howard et al.
1998) are all pre-2019 and were **not** independently re-extracted in our new (Santos)
dataset. They therefore enter the combined data **only** through the Davies (continuous)
and Jones (binary) datasets, so every edit below acts on legacy rows only — the new
extraction is untouched.
:::

## Identifying the contentious effect sizes

We read the re-extraction file produced by the validation chapter. Each of its rows is
one effect size that we returned to the source PDF for, and it carries three things side
by side: the **re-extracted** value (independent, design-faithful), the **Jones** value,
and the **Davies** value. From this file alone we can locate every row that needs editing
in the two combined datasets — by `identifierEffectSizeID`, never by guesswork.

```{r}
#| label: load_reextraction
#| message: false
#| warning: false
reext_path <- here::here("outputs", "3_original_dataset_crosscheck", "reextraction_flagged_7studies.csv")
reext <- readr::read_csv(reext_path, show_col_types = FALSE)

# Conversion constant: lnOR -> Hedges' d (logistic-to-normal; Borenstein et al. 2009).
# Matches the validation chapter; the small-sample (J) correction is ~1 at these
# sample sizes and is omitted so the numbers agree with that chapter.
LNOR_TO_D  <- sqrt(3) / pi          # ~ 0.5513
VAR_LNOR_TO_VAR_D <- 3 / pi^2       # = LNOR_TO_D^2 ~ 0.3040

reext_map <- reext %>%
    transmute(
        study          = identifierStudyId,
        # Jones effect-size code in the dataset has no slash (e.g. "APP2000THE/1" -> "APP2000THE1")
        jones_es_id    = str_remove(compare_jones_code, "/"),
        # Davies effect-size number, parsed from e.g. "Davies study 3 (3.1)" -> "3.1"
        davies_es_id   = str_match(compare_davies_es, "\\(([0-9]+\\.[0-9]+)")[, 2],
        davies_study_no = str_match(compare_davies_es, "study ([0-9]+)")[, 2],
        species_phylo  = str_squish(as.character(taxonomySpecies)),
        # Re-extracted values
        reext_lnOR     = log(effectSizeCalculatedEstimate),
        reext_var_lnOR = effectSizeCalculatedVariance,
        reext_d        = log(effectSizeCalculatedEstimate) * LNOR_TO_D,
        reext_var_d    = effectSizeCalculatedVariance * VAR_LNOR_TO_VAR_D,
        # Prior values (for the audit table)
        jones_lnOR     = compare_jones_lnOR,
        jones_var      = compare_jones_VlnOR,
        davies_d       = compare_davies_hedges_d,
        davies_var     = compare_davies_variance
    )

# The set of contentious studies, expressed in each dataset's own identifiers.
davies_studies_flagged <- sort(unique(as.integer(reext_map$davies_study_no)))   # Davies study numbers
jones_papers_flagged   <- sort(unique(str_extract(reext_map$jones_es_id, "^[A-Za-z]+[0-9]{4}")))  # Jones paper prefixes

# Jones rows the validation identified as over-counts to be dropped entirely:
#  * DUG1992REV1  — spurious third value (reproduces from no experiment pair)
#  * GIE2018MAT1/2 — duplicate of the GIE2018USI contrast (over-count)
jones_drop_overcount <- c("DUG1992REV1", "GIE2018MAT1", "GIE2018MAT2")
```

# ===========================================================================

or_leaveout <- db_or %>%
    filter(!(Dataset_Source == "Original (Jones & DuVal 2019)" &

# (2) CORRECTED: (a) drop the over-count rows; (b) replace lnOR & variance of the
#     re-extracted codes with the design-faithful values; leave all other Jones rows
#     (e.g. the legitimate, un-re-extracted APP2000THE2 baseline contrast) untouched.
or_replacements <- reext_map %>% select(identifierEffectSizeID = jones_es_id, reext_lnOR, reext_var_lnOR)

## Re-fitting the combined models

We use the **same** multilevel model as the overall-effect chapter: `rma.mv` with a
variance–covariance matrix (`vcalc`, within-study $\rho = 0.5$), random intercepts for
study, effect size, and species, and a phylogenetic random effect supplied with the
species correlation matrix (it falls back to the non-phylogenetic structure if a species
is not in the matrix).

```{r}
#| label: fit_function
#| message: false
#| warning: false
fit_overall_mv <- function(dat, cluster_var = "identifierStudyId",
                           obs_var = "identifierEffectSizeID", rho = 0.5,
                           phylo_cor = NULL, phylo_var = "species_phylo") {
    dat2 <- dat %>% filter(!is.na(yi), !is.na(vi), vi > 0)
    if (nrow(dat2) == 0) return(NULL)

    VCV <- metafor::vcalc(vi = dat2$vi, cluster = dat2[[cluster_var]], obs = dat2[[obs_var]],
                          rho = rho, data = dat2)

    random_list <- list(~ 1 | identifierStudyId, ~ 1 | identifierEffectSizeID, ~ 1 | taxonomySpecies)
    R_arg <- NULL
    if (!is.null(phylo_cor) && phylo_var %in% colnames(dat2)) {
        sp <- unique(as.character(dat2[[phylo_var]]))
        if (length(setdiff(sp, rownames(phylo_cor))) == 0) {
            dat2[[phylo_var]] <- factor(as.character(dat2[[phylo_var]]))
            Csub <- phylo_cor[sp, sp, drop = FALSE]
            random_list <- c(random_list, list(stats::as.formula(paste0("~ 1 | ", phylo_var))))
            R_arg <- stats::setNames(list(Csub), phylo_var)
        }
    }
    mod <- metafor::rma.mv(yi = dat2$yi, V = VCV, random = random_list, R = R_arg,
                           test = "t", method = "REML", sparse = TRUE, data = dat2)
    list(data = dat2, model = mod)
}

# Fit all six models
fits <- list(
    hedges_main      = fit_overall_mv(db_hedges,        phylo_cor = phylo_cor_hedges),
    hedges_leaveout  = fit_overall_mv(hedges_leaveout,  phylo_cor = phylo_cor_hedges),
    hedges_corrected = fit_overall_mv(hedges_corrected, phylo_cor = phylo_cor_hedges),
    or_main          = fit_overall_mv(db_or,            phylo_cor = phylo_cor_or),
    or_leaveout      = fit_overall_mv(or_leaveout,      phylo_cor = phylo_cor_or),
    or_corrected     = fit_overall_mv(or_corrected,     phylo_cor = phylo_cor_or)
)
```

# Pull a tidy one-row summary from each fitted model
summarise_fit <- function(fit, class_lab, version_lab) {
    if (is.null(fit)) return(NULL)
    m <- fit$model
    i2 <- tryCatch(sum(orchaRd::i2_ml(m)["I2_Total"]), error = function(e) NA_real_)
    tibble(
        Class = class_lab, Version = version_lab,
        k = m$k, Studies = length(unique(fit$data$identifierStudyId)),
        Estimate = as.numeric(m$b[1]), SE = m$se[1],
        CI_low = m$ci.lb[1], CI_high = m$ci.ub[1],
        t = m$zval[1], p = m$pval[1], I2 = i2
    )
}

### Comparison plot

We display each version as its own orchard trunk — the same plot format used elsewhere in
this book. Each row is one model version (Main, Leave-out, Corrected); the bubbles are that
version's raw effect sizes (scaled by precision), the central marker is the pooled mean with
its 95% confidence interval, and the thin whisker is the 95% prediction interval. The
`Mean [95% CI]` and `k (n)` annotations sit beside each trunk.

```{r}
#| label: orchard_helpers
#| message: false
#| warning: false
ver_levels <- c("Main", "Leave-out", "Corrected")
ver_cols   <- c("Main" = "#444444", "Leave-out" = "#D55E00", "Corrected" = "#0072B2")

# Build one orchard "trunk" (an orchaRd mod_results object) from a fitted model and
# relabel it with the version name, mirroring the multi-trunk approach in overall_effect.qmd.
mk_trunk <- function(fit, name) {
    r <- orchaRd::mod_results(fit$model, group = "identifierStudyId")
    class(r) <- "list"
    r$mod_table$name <- name
    r$data$moderator <- name
    class(r) <- c("orchard", "data.frame")
    r
}

# Place "Mean [95% CI]" above the existing "k = .. (..)" label on each trunk
# (identical helper to the overall-effect chapter).
add_ci_labels <- function(plot_obj, orchard_results) {
    est  <- sprintf("%.2f", orchard_results$mod_table$estimate)
    ci_l <- sprintf("%.2f", orchard_results$mod_table$lowerCL)
    ci_u <- sprintf("%.2f", orchard_results$mod_table$upperCL)
    orig_labels <- plot_obj$layers[[6]]$aes_params$label
    new_labels <- character(0)
    for (i in seq_along(orig_labels)) {
        nums <- regmatches(orig_labels[i], gregexpr("[0-9]+", orig_labels[i]))[[1]]
        k_text <- if (length(nums) >= 2) paste0("k = ", nums[1], " (", nums[2], ")") else orig_labels[i]
        new_labels[i] <- paste0(est[i], " [", ci_l[i], ", ", ci_u[i], "]\n", k_text)
    }
    plot_obj$layers[[6]]$aes_params$label <- new_labels
    plot_obj$layers[[6]]$geom_params$parse <- FALSE
    plot_obj$layers[[6]]$aes_params$hjust <- "right"
    plot_obj$layers[[6]]$aes_params$vjust <- 0.5
    plot_obj$layers[[6]]$aes_params$size <- 3.0
    plot_obj
}

# Assemble a three-trunk orchard plot (Main / Leave-out / Corrected) for one class.
sensitivity_orchard <- function(fit_main, fit_leave, fit_corr, xlab, title) {
    t_main <- mk_trunk(fit_main,  "Main")
    t_leav <- mk_trunk(fit_leave, "Leave-out")
    t_corr <- mk_trunk(fit_corr,  "Corrected")
    sub <- orchaRd::submerge(t_main, t_leav, t_corr)

    p <- orchaRd::orchard_plot(sub, xlab = xlab, flip = TRUE) +
        ggplot2::scale_colour_manual(values = ver_cols, name = "Dataset version") +
        ggplot2::scale_fill_manual(values = ver_cols, name = "Dataset version") +
        ggplot2::ggtitle(title) +
        ggplot2::theme(
            legend.position = "none",
            plot.title = ggplot2::element_text(face = "bold", hjust = 0.5)
        )
    add_ci_labels(p, sub)
}
```

## Interpretation

```{r}
#| label: interp_values
#| echo: false
getv <- function(cl, ver, col) res[[col]][res$Class == cl & res$Version == ver]
hm <- getv("Hedges' g", "Main", "Estimate"); hl <- getv("Hedges' g", "Leave-out", "Estimate"); hc <- getv("Hedges' g", "Corrected", "Estimate")
om <- getv("Log OR", "Main", "Estimate");    ol <- getv("Log OR", "Leave-out", "Estimate");    oc <- getv("Log OR", "Corrected", "Estimate")
hm_p <- getv("Hedges' g", "Main", "p"); hl_p <- getv("Hedges' g", "Leave-out", "p"); hc_p <- getv("Hedges' g", "Corrected", "p")
om_p <- getv("Log OR", "Main", "p");    ol_p <- getv("Log OR", "Leave-out", "p");    oc_p <- getv("Log OR", "Corrected", "p")
sig <- function(p) if (is.na(p)) "indeterminate" else if (p < 0.05) "statistically significant" else "non-significant"
```

**Continuous (Hedges' *g*).** The combined mean is $`r fmt(hm, 2)`$ in the **Main**
analysis (`r sig(hm_p)`), $`r fmt(hl, 2)`$ when the contentious studies are **left out**
(`r sig(hl_p)`), and $`r fmt(hc, 2)`$ when they are **corrected** (`r sig(hc_p)`). Because
the validation showed the Davies errors mostly *inflate* the effect, the leave-out and
corrected means are expected to move toward — not away from — zero relative to Main;
the size of that shift is the practical measure of how much the headline continuous
estimate leaned on the flagged extractions.

**Binary (log odds ratio).** The combined mean is $`r fmt(om, 2)`$ in **Main**
(`r sig(om_p)`), $`r fmt(ol, 2)`$ **leaving out** the contentious papers (`r sig(ol_p)`),
and $`r fmt(oc, 2)`$ when **corrected** (`r sig(oc_p)`). On the binary side the
corrections act mainly on *variances* (removing the phantom-control inflation) and remove
two over-counted rows, so the point estimate is expected to move less than the continuous
one; the main effect there is on precision and weighting rather than on the location of
the mean.

**Reading the three together.** The **Leave-out** column is the conservative bound (trust
nothing flagged); the **Corrected** column is the best estimate (keep the studies, fix the
errors). If both fall close to **Main** and keep the same sign and significance, the
overall conclusion is robust to the extraction problems the validation uncovered. If
**Main** sits apart from the other two, the headline estimate depended materially on the
contentious effect sizes, and the **Corrected** value should be preferred as the
bias-reduced summary.

::: {.callout-warning}
## What this analysis does and does not show
This is a sensitivity analysis of the **combined overall mean** to the seven studies the
cross-dataset validation flagged. It does **not** re-audit the rest of the literature, and
it inherits every other modelling choice from the overall-effect chapter (the multilevel
structure, the within-study $\rho = 0.5$, and the lnOR$\leftrightarrow$*d* scale
conversion). The "Corrected" edits are exactly those documented in the
[validation chapter](../3_original_dataset_crosscheck/extraction_validation.qmd) and listed
in the edit map above; no other values were changed.
